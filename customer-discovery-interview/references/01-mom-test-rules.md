# Mom Test 操作規則

核心命題：連你媽都無法對你說謊的問題，才是好問題。人會為了不傷你的心而說謊，所以問題設計要讓「說謊比說實話更費力」。

## 三規則操作化

### 規則一：談他們的生活，不談你的點子

- 問題探索階段的訪談，全程不揭露解法。開場定位話術：「我今天不是來推銷任何東西，我在研究大家怎麼處理 X 這件事，想聽你的實際經驗，你的日常做法對我最有價值。」
- 對方追問「所以你們到底要做什麼？」時：「等等一定跟你講。但我先講了，你的答案就會被我污染，先讓我聽你原本的版本，最後五分鐘留給你問我。」
- 一旦揭露點子，之後的回答全部降級為 C 級口頭證據，因為對方開始「幫你想」而不是「講自己」。
- 例外：solution validation 階段可以展示 prototype，但順序是「先問完過去行為，再展示，再觀察反應」，不能反過來。

### 規則二：問過去的具體行為，不問未來假設

- 人對未來的自己過度樂觀，對過去的行為無法造假。任何「你會不會」都能改寫成「你上次有沒有」。
- 具體化三件套，卡住就輪流用：
  1. 「上一次發生是什麼時候？」
  2. 「當時的情況是怎樣？」
  3. 「你接著做了什麼？」
- 錨定具體事件，不接受平均值：「你通常都怎麼做」得到的是自我美化的摘要，改問「講一個最近的具體例子」。
- 金額、時間、次數追到數字為止：「花很多時間」→「大概多久？上週花了幾小時？」
- 判斷規則：受訪者講不出具體事件的細節（日期、場景、金額），代表這件事對他不重要，或根本沒發生過。

### 規則三：少說多聽

- 你的說話佔比壓在兩到三成以下。訪談不是對談，是挖掘。
- 沉默技巧：對方講完停頓時，心裡數三秒再開口。最有料的補充常出現在你忍住的那三秒。
- 覆述確認而非解讀：「所以你的意思是，你後來乾脆自己用 Excel 拉了一份，我理解對嗎？」錯了對方會糾正你，這也是資料。
- 三個不要：不要示範你很聰明、不要糾正對方的「錯誤用法」、不要替你的產品辯護。對方誤解你的產品也是資料——真實用戶就會這樣誤解。

## 訪談前準備：三個大問題

每場訪談前先寫下「這場最想學到的三件事」，跟團隊對齊後才進場。規則：

- 三個問題要是「答案可能讓你害怕」的問題：這問題到底存不存在、他們現在付多少錢給誰、採購會卡在哪一關。答案不痛不癢的問題不配佔名額。
- 每個團隊成員對「最大的未知」認知常常不同，訪前十分鐘對齊，比訪後吵架便宜。
- 訪談中可以偏題，但收尾前檢查三個問題是否都拿到了事實或明確的「問不到」。

## Keep it casual：非正式訪談

問題探索階段不一定要正式約訪。正式會議有三個成本：對方進入「被推銷」防衛模式、你進入「必須表現」模式、排程慢。

- 展場、聚會、排隊、既有會議的前五分鐘，都能塞進一兩個 Mom Test 問題：「欸你們現在報表都怎麼弄的？」
- 非正式對話拿到的事實跟正式訪談等值，一樣要記錄出處與情境。
- 進入 solution validation、需要 commitment 測試時才升級成正式會議——要人掏出時間、聲譽、金錢，本來就該正式。

## Zoom out / Zoom in：問題的高度控制

- 開太窄（直接問某功能細節）會拿到規格意見；開太寬（「聊聊你的工作」）會聊成傳記。
- 順序：先 zoom out 確認這個問題在對方生活裡的位置——「這件事在你所有麻煩裡排第幾？前面還有哪些更煩的？」排不進前三的問題不值得做。
- 確認位置後再 zoom in 到具體事件重建細節。
- 常見翻車：受訪者抱怨 X，你立刻栽進 X 的細節，訪完才發現 X 在他心中根本排不進前十。

## 好壞問題對照表

| 壞問題 | 改成 | 為什麼 |
|---|---|---|
| 你覺得這是個好點子嗎？ | 你現在都怎麼處理 X？帶我走一次上次的流程。 | 點子好壞只有市場能回答，人只會客氣 |
| 你會買這樣的產品嗎？ | 你上次遇到這問題，花了多少錢或時間解決？ | 未來承諾免費，過去付出才是真的 |
| 你願意付多少錢？ | 你現在為了這件事每個月已經在花多少？ | 現有花費是唯一可信的定價錨點 |
| 如果有工具能 X，你會用嗎？ | 你上一次需要 X 是什麼時候？當下你怎麼做？ | 「會用」不需成本，回憶造不了假 |
| 你常遇到 X 問題嗎？ | 上一次遇到 X 是什麼時候？當時發生什麼事？ | 「常」是印象，事件是事實 |
| X 功能對你重要嗎？ | 你有沒有為了 X 做過什麼？找工具、問人、自己土法煉鋼？ | 重要性看行動不看嘴巴 |
| 這問題對你嚴重嗎？ | 這問題發生之後，你接下來做了什麼？ | 什麼都沒做＝不嚴重，問卷分數會騙人 |
| 你未來會想要 X 嗎？ | 你過去有沒有試著自己解決過？怎麼做的？ | workaround 的存在證明需求存在 |
| 我們打算做 X，你覺得如何？ | 先不講我們要做什麼——你目前這塊的流程長怎樣？ | 先講點子就污染整場訪談 |
| 你的預算大概多少？ | 上次採購類似的東西，錢從哪個預算出？誰簽核？ | 抽象預算答不準，實際採購路徑答得出來 |
| 你最想要什麼功能？ | 上次用（現有工具）時，哪一步最讓你想罵人？ | 用戶給的是痛點不是規格，功能要自己設計 |
| 你會推薦給朋友嗎？ | 你有跟誰抱怨或聊過這個問題？當時怎麼講的？ | 已發生的口碑行為才算數 |
| 這個價格可以接受嗎？ | 你為了省掉這個麻煩，付過最貴的一次是多少？ | 接受度是社交回答，付款紀錄是行為 |
| 你每天會打開來用嗎？ | 你現在每天固定做哪些事來處理這件事？ | 現有習慣才是頻率的證據 |
| 如果免費你會用嗎？ | 你現在在用的免費方案是什麼？它哪裡讓你忍到現在？ | 「免費會用」毫無資訊量，連問都不要問 |
| 你覺得市場上還缺什麼？ | 你上次找解法時搜尋了什麼關鍵字？最後為什麼沒買？ | 缺口要從真實搜尋與放棄行為回推 |

## 三種假訊號與現場轉化話術

### 假訊號一：讚美

徵兆：「聽起來不錯」「這點子很棒」「一定很多人需要」。讚美是資訊量為零的社交潤滑劑，而且最危險——它讓你爽。

轉化（deflect 回具體行為）：
- 「謝謝——不過先別管我們的東西。你上次自己遇到這狀況，實際上是怎麼處理的？」
- 「你說很多人需要，你身邊有誰真的被這個困擾？他現在怎麼辦？」

記錄規則：逐字稿裡的讚美一律不編碼為正向訊號，只標記「此處已 deflect / 未 deflect」。

### 假訊號二：空泛承諾

徵兆：「上線我一定用」「我以後會買」「到時候找我」。沒有具體時間、具體代價的承諾都是客氣話。

轉化（當場要求 commitment）：
- 「太好了。那我們下週安排三十分鐘讓你試 beta 版好嗎？你行事曆哪天方便？」
- 「如果你真的有這需求，方便介紹我認識你們團隊裡實際負責這塊的人嗎？」
- 對方開始閃躲（「再看看」「最近比較忙」）＝承諾作廢，這本身就是有效資料。

### 假訊號三：突然給點子

徵兆：「你們應該加一個 X 功能」「可以跟 Y 整合啊」。點子是資料的線索，不是規格需求。

轉化（記下，回挖動機）：
- 「有意思，我記下來。你會想到這個，是因為之前碰過什麼狀況嗎？」
- 「如果有了這功能，你現在哪件事會變不一樣？」
- 挖到背後的 job 之後就放下，不承諾做，也不當場評估可行性。

## Commitment 與推進測試

### 三種 commitment currencies

- 時間：花具體時間試用、參加下次會議、幫你跑內部流程
- 聲譽：把你介紹給同事、老闆、其他潛在客戶；公開背書
- 金錢：預付、訂金、LOI、簽 pilot 合約

### 推進階梯（由弱到強）

1. 同意收後續消息（幾乎無成本，勉強算）
2. 明確約好下次會議，日期在行事曆上
3. 花時間試 prototype 並回饋具體意見
4. 介紹決策相關的其他人（動用聲譽）
5. LOI、預付、簽 pilot（動用金錢）

### 判斷規則

- 每場訪談結束時，要嘛拿到具體事實（可編碼的過去行為），要嘛拿到 commitment 推進。兩者皆無＝這場白開，而且對方多半在敷衍。
- 「會議結果很順利、對方人很好」不是結果。問自己：他付出了什麼？
- solution validation 階段，連續多場拿不到任何 commitment，優先懷疑問題不夠痛，而不是話術不夠好。

## B2B 訪談的多角色注意

B2B 的「顧客」至少拆四個角色，各自的 push 和 anxiety 完全不同：

- 用戶：每天操作的人。在乎省時省力，痛點最具體，但沒有錢也沒有票。
- 買家：出預算的人。在乎 ROI、能不能跟老闆交代。
- 決策者：拍板的人。在乎風險、政治、跟現有策略的相容。
- 守門人：採購、資安、法務、IT。不會說好，只會說不行。

角色路徑問法：
- 「上次你們導入類似的工具，是誰先提出來的？中間經過誰？最後誰簽的？」
- 「這種金額的採購要跑什麼流程？通常多久？曾經卡在哪一關？」
- 「有沒有哪個工具是大家很想要、最後卻沒過的？卡在哪？」

執行規則：
- 一個案子至少涵蓋用戶與買家兩種角色，只訪用戶會高估需求（他們不用付錢）。
- 訪談對象自稱「我可以決定」時，用上面的路徑問法驗證，多數人高報自己的決策權。
- 守門人的否決條件（資安審查、個資、既有合約）要在 design 階段就列入研究問題，這是 B2B 最常見的死因。

## 訪談中紅燈自查

出現以下任一情況，立刻停下拉回：

- 你已經連續講超過一分鐘 → 你在 pitch，不是在訪談
- 你剛問了一句「如果……你會……嗎」 → 改成「上次……你做了什麼」
- 對方連續講的都是「應該」「大概」「我覺得一般人會」 → 拉回他本人的具體事件
- 你心裡在為對方的讚美高興 → 標記假訊號，deflect
- 筆記全是形容詞沒有數字和事件 → 開始追金額、時間、次數
