# Threads Viral Growth — 爆文操作 skill

## 這個 skill 在做什麼

幫使用者產出「真的會在台灣 Threads 上被推、被留言、被轉發」的貼文。不是寫漂亮文案、不是想 idea，是直接交付**可以複製貼上發布**的草稿，加上發文前的趨勢佐證、發文後的留言互動劇本。

核心信念三條，所有產出都不得違反：

1. **趨勢先於創意**。沒做趨勢偵測就動筆寫貼文，是用昨天的素材打今天的演算法。哪怕只是 5 分鐘的快查，也必須做。
2. **互動 > 觸及**。Threads 的演算法核心是「能不能引發討論」，不是「有沒有讚」。每篇貼文都要設計留言鉤。互動權重順序：**發文分享/引用 > 留言 > 收藏 > 讚**。回覆鏈權重佔觸及 70% 以上。
3. **人話 > 文案**。寫的是「TA 心裡早就想講，但沒人幫他講出來的那句話」，不是雞湯、不是教學、不是業配。

> 「Threads 演算法偏愛的不是長篇大論，而是帶有情緒或獵奇色彩的短文。」—— 媒體人觀察
> 「真正困難的是寫人話，搞笑可以複製，但觀點很難複製。」—— 行銷實測

## 何時應該觸發這個 skill

明確訊號：使用者提到「Threads / 脆」 + 想寫東西或經營帳號。
隱性訊號：使用者談到品牌、課程、店家想做社群，且還沒指定平台，但語氣偏向短文、年輕、情緒導向 → 主動建議用 Threads，並啟動 skill。

如果使用者只是隨口提到「想紅」「想被看見」而沒指出平台，可以詢問是否要用 Threads 為主，但不要強推。

## 主流程（必須按順序）

```
Step 1  品牌/任務定位確認   ← 5-10 分鐘對齊
Step 2  趨勢偵測（必做）    ← 不可跳過
Step 3  題材篩選            ← 品牌 × 趨勢交集
Step 4  情緒定錨 + Hook 設計
Step 5  貼文成型（爆文五件套）
Step 6  時間排程 + 發文後 15 分鐘劇本
Step 7  數據回饋與優化
```

**禁止行為**：跳過 Step 2 直接寫貼文。即使使用者很急、即使主題看起來很顯然，都要至少做一次 5 分鐘快查（web_search 兩三個關鍵字也行）。理由：Threads 是高度即時的平台，3 天前的梗今天可能已經死了，你寫出來會像長輩文。

## Step 1｜定位確認

問清楚（如果使用者沒提供）：

- **發文主體是誰**：個人創作者 / 品牌 / 學校系所 / 店家 / 課程作業？
- **目標受眾是誰**：高中生、大學生、上班族、媽媽族、特定興趣社群？年齡層、性別、生活情境要具體。
- **品牌人格**：用 3 個形容詞描述（例：誠實的、有點壞、社群感、AI 感、年輕感、實戰感）。
- **本次任務目標**：漲粉？引發討論？導流？品牌印象？課程交付？
- **禁區**：什麼不能講（公司未公開的事、競品名稱、政治立場等）。

如果是課程作業（像北市大行銷系 Final Task 這種），把作業要求的交付項目列出來對齊。

定位輸出範例（簡短版本即可，不用寫一份提案）：

```
主體：北市大行銷系
TA：高中生（17-18 歲，正在選系階段）
人格：實戰、有點壞、社群感、不端著
目標：讓高中生記住北市大行銷系、引發留言、產生「想念這裡」的衝動
禁區：不貶低其他大學、不過度官腔
```

## Step 2｜趨勢偵測（**必做**）

目標：在動筆前花 10-15 分鐘，搞清楚現在 Threads 上「對應你 TA 的圈層」正在聊什麼。

**4 個資料來源，按順序查**：

1. **Threads 平台內趨勢話題**：請使用者打開 Threads → 右上角搜尋 → 看當前全台前 5 大熱門話題。如果你是 AI 沒辦法直接看 Threads，請使用者截圖或描述給你，或請使用者點開「超夯」標籤的貼文。
2. **web_search 即時話題**：用 `web_search` 查 `Threads 熱門 [TA 圈層關鍵字] [近期月份]`。例：「Threads 熱門 大學生 11月」、「Threads 大學申請 討論」。
3. **競品/同類帳號最近 7 天爆文**：請使用者列 3-5 個同類帳號（或自己用搜尋找），看他們最近哪幾則互動數明顯較高。觀察 hook、句型、情緒。
4. **跨平台時事**：看 PTT、Dcard、新聞頭條有沒有什麼話題可能延伸到你的 TA 圈層。

**輸出格式**：把研究做成一份**趨勢備忘**，給使用者看，再進下一步。

```
今日趨勢備忘（YYYY/MM/DD）
─────────────────────────
Threads 平台趨勢話題：
  ① [話題A] — 為什麼夯：…
  ② [話題B] — 為什麼夯：…

對應 TA 的圈層討論：
  • [關鍵討論點 1]
  • [關鍵討論點 2]

可借用的句型/梗：
  • 「____」這種 hook 最近常見，原因：…

可挑戰/反向操作的點：
  • 大家都在說 X，但其實可以說 Y…
```

**完整 SOP 與工具清單**：詳見 `references/01-trend-research.md`

## Step 3｜題材篩選

從趨勢備忘裡篩出 **3 個候選題材**，每個題材必須通過下面四道閘：

| 閘門 | 通過條件 |
|---|---|
| **TA 痛點** | TA 看到會點頭、想罵髒話、或想截圖傳給朋友 |
| **品牌連結** | 寫完後讀者能聯想到你的品牌 / 系所 / 主體，而不是其他人都能寫 |
| **可討論** | 至少 5 種留言可以預想（贊同、反對、補充、分享經驗、開玩笑） |
| **不踩雷** | 不政治對立、不歧視、不嘲諷弱勢、符合品牌禁區 |

如果候選題材通過 4 道閘的不到 3 個，**回 Step 2 再找**，不要將就。將就出來的貼文一定不會爆。

## Step 4｜情緒定錨 + Hook 設計

### 4.1 選一個主導情緒

每則貼文只能有**一個主導情緒**。混合情緒的貼文會稀釋傳播力。

六大主導情緒（爆文最常用）：

| 情緒 | 觸發行為 | 適用 |
|---|---|---|
| 共鳴 | 留言「+1」「就是我」 | 受眾痛點、生活吐槽 |
| 憤怒 | 引用反擊、長留言 | 不公、雙標、亂象 |
| 幽默 | 轉發、截圖 | 自嘲、產業內梗 |
| 驚訝 | 留言「真假」「沒想到」 | 反差、冷知識、不為人知的事實 |
| 焦慮 | 收藏、搜尋更多 | 競爭、限時、稀缺 |
| 溫暖 | 分享給朋友、留言感謝 | 故事、被理解、被支持 |

**情緒設計細節**：詳見 `references/05-emotion-framework.md`

### 4.2 寫 Hook

Hook = 貼文第一行（最多前兩行），決定貼文 70% 的生死。

**Hook 五大類型**（先確定要用哪一種）：

1. **疑問型** — 「真的有人 [X] 嗎？」「你有沒有發現 [Y]？」
2. **吐槽型** — 「最累的不是 [X]，是 [Y]」「[X] 根本是騙人的」
3. **共鳴型** — 「很多人以為 [X]，但其實 [Y]」「我們都經歷過 [Z]」
4. **爭議型** — 「[多數人觀點] 是錯的，我來說一個你不愛聽的」
5. **反差型** — 「我以為 [X]，直到 [Y]」「沒想到 [意外結局]」

**Hook 完整模板庫（50 個範例）**：詳見 `references/03-hook-library.md`

**Hook 自檢三問**：
1. 一個陌生人在動態牆滑到，會停下來嗎？
2. 第一行有沒有「鉤點」（疑問、衝突、反差、痛點之一）？
3. 是不是 25 字以內？（超過很容易被滑過）

## Step 5｜貼文成型（爆文五件套）

每則貼文必須有這五件套：

```
┌────────────────────────────────────────┐
│  ① Hook          ← 前一兩行 ≤ 25 字     │
│  ② 鋪陳          ← 2-4 行，短句、口語化 │
│  ③ 觀點/反差     ← 一句核心主張         │
│  ④ 留言鉤        ← 開放式問句 / 立場選擇│
│  ⑤ 標籤（可選）  ← 一篇最多一個         │
└────────────────────────────────────────┘
```

### 寫作硬規則

| 規則 | 為什麼 |
|---|---|
| 全文 ≤ 350 字（理想 150-250 字） | 演算法偏好可滑完的短文 |
| 短句、空行多 | Threads 是手機平台，密集文字會被秒滑 |
| 不用書面語 | 用「啊」「欸」「真的」「但是」這種口語 |
| 不放外部連結在內文 | 觸及會被降權，連結放第一則留言 |
| 一篇最多一個 hashtag | 多 hashtag 反而降權 |
| 不寫業配感 | 用「我發現…」「我朋友…」「我自己也…」這種敘事口吻 |

### 結構模板（5 種，可直接套）

詳見 `references/04-post-formulas.md`。**這個檔每次寫貼文都要讀**，裡面有：

- 共鳴型模板（最常爆）
- 爭議型模板（留言最多）
- 故事型模板（漲粉最有效）
- 清單型模板（收藏率最高）
- 反問型模板（觸及最廣）

### 貼文自檢清單（送出前必查）

每寫完一則貼文，跑一次這 7 問：

1. Hook 是不是 25 字內，有沒有鉤點？
2. 主導情緒是哪一個？只有一個嗎？
3. 留言鉤在哪裡？讀者會怎麼回？預想 5 種留言。
4. TA 看到會不會點頭/想截圖/想反駁？三選一以上。
5. 有沒有業配感、官腔、長輩文味道？有就改。
6. 連結是不是在內文裡？是的話搬到第一則留言。
7. 字數是不是超過 350？超過就砍。

## Step 6｜時間排程 + 發文後 15 分鐘劇本

### 6.1 台灣 Threads 最佳發文時間

依 QSearch 分析 2025 年全年 3,600 萬則台灣貼文：

| 時段 | 互動指數 | 適合貼文類型 |
|---|---|---|
| **週四 20:00–21:00** | ★★★★★（最佳） | 重磅內容、想引爆的主題 |
| 週二–週四 20:00–22:00 | ★★★★ | 一般主題 |
| 平日 07:00–09:00 | ★★★ | 通勤輕量內容、新聞延伸 |
| 平日 12:00–13:00 | ★★★ | 午休吐槽文、輕鬆共鳴 |
| 週末 14:00–17:00 | ★★ | 生活感、放鬆內容 |
| 週末晚上 | ★ | 互動量明顯下降，不建議重要內容 |

**完整時間策略**：詳見 `references/06-posting-timing.md`

### 6.2 發文後關鍵 15 分鐘劇本

Threads 演算法會在發文後初期（約 30–60 分鐘）觀察互動率，決定是否擴大推播。**前 15 分鐘是黃金窗口**。

劇本：

```
T+0   發文
T+1   立刻自留言補充內容（或放連結）
T+3   請 2-3 個朋友/同伴去留言，留有觀點的話，不要「讚」「+1」
T+5   你自己回那 2-3 個留言，回得有內容
T+10  到一兩個熱門帳號的近期貼文下，留一則「觸」的留言（海巡法）
T+15  檢查貼文觸及，如果起飛了，準備寫第二則 / 補充串文
```

**完整互動策略**：詳見 `references/07-engagement-tactics.md`

## Step 7｜數據回饋

每則貼文發出 48 小時後，紀錄：

- 觸及數
- 讚 / 留言 / 引用 / 分享 / 轉發
- 哪則留言最熱
- 演算法有沒有把它推給非粉絲

用 `assets/analytics-template.md` 模板紀錄，至少累積 10 則後可以開始找出規律：哪種 Hook 對你 TA 最有效、哪個時段你的 TA 最活躍、哪種情緒勝率最高。

## 圖卡設計（如果需要）

Threads 的圖卡風格高度一致：

- **黑底白字** or **白底黑字**，極簡
- **大字排版**，一張卡片只說一句話
- **手機豎屏** 4:5 或 1:1
- **底部有品牌簽名 + 一句小字補充**
- **要好截圖、好被「轉發給朋友」**

詳見 `references/08-card-design.md`，裡面有 Canva / Affinity / AI 生圖的 prompt 模板。

## 給使用者的最終交付（除非另有指示）

每次完整跑完流程，輸出包：

1. **趨勢備忘**（Step 2 的結果）
2. **題材篩選結果**（為什麼選這個題材）
3. **可直接發布的貼文草稿**（含 Hook、本文、留言鉤、自留言內容）
4. **預想留言情境** 5 種（哪些人會用什麼角度留言）
5. **建議發文時段**
6. **發文後 15 分鐘劇本**
7. **配套圖卡 prompt**（如果要圖卡）

⚠️ 不要交付「五個提案請選一個」這種模糊輸出。每次就交付 1-2 則最完整的，使用者覺得不對再 iterate。**精度比廣度重要**。

## 最後一句

爆文沒有公式，但有規律。這個 skill 把規律拆成 SOP，是為了**降低不爆的機率**，不是保證每篇都爆。
真正爆的關鍵還是：你有沒有真的觀察你的 TA、有沒有講出他們想講但沒講出來的那句話。
工具與流程是助力，**洞察才是核心**。
