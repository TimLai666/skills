# 創意發想

## 主題定位

《解決問題的商業框架圖鑑》解釋，激發創意的過程分為發散與收斂：發散是蒐集、拆解資訊，不設限地拋出各種想法；收斂是把發散後的點子整理、分類、結合，將無形化為有形。與創意相關的模型，至少具備發散或收斂其中一種功能。先累積一定的量，才能淬煉出新奇的看法。

套用到任何任務上：動手前先刻意發散，多想幾種可能，再刻意收斂成可用的方向。不要拿第一個想法就定案。

## 六頂思考帽：輪流戴上六種觀點，跳脫慣性立場

腦力激盪時，為避免團隊成員的思考受限於差不多的背景與想法，思考大師愛德華・狄波諾（Edward de Bono）提出六頂思考帽，各用一種顏色代表一種觀點。成員必須跳脫自己的立場，依照被主持人分配到的角色發言；主持人也可以透過變換成員的帽色加強討論熱度，集結各式各樣的創意。

原方法是團隊工具。Agent 使用時，優先用 subagent 還原團隊：自己擔任藍帽主持人，其餘五頂帽各開一個獨立 context 的 subagent 扮演。每個 subagent 只拿到任務說明、自己的帽色角色與必要的前置產出，看不到其他帽子的發言，觀點才不會互相污染。環境不支援 subagent 時，才退回單人輪流戴帽。六頂帽同時涵蓋發散（綠帽衝量）與收斂（藍帽彙整）。

| 帽色 | 觀點 | 核心提問 |
| --- | --- | --- |
| 藍帽（主持人） | 綜觀、管理 | 確立主題和任務；彙整、決定方案 |
| 白帽 | 中立、客觀 | 參考資訊有哪些？有哪些實際案例？ |
| 黃帽 | 積極、樂觀 | 優勢是什麼？如何做才能實現？ |
| 黑帽 | 消極、否定 | 可能的風險？競爭對手（外部阻力、替代選項）有誰？ |
| 綠帽 | 創造、革新 | 怎麼做更好？有無替代方案？ |
| 紅帽 | 情感、主觀 | 直覺反應為何？喜歡或討厭？ |

### 操作步驟

預設輪帽順序如下。藍帽是主持人，由主 agent 擔任，開場與收場都是它；白、綠、黃、黑、紅五帽在環境支援時各開一個 subagent 扮演，每個 subagent 的提示只含：該帽的角色與核心提問、任務說明、該帽需要的前置產出，不含其他帽子的發言。

1. **藍帽開場（主 agent）** — 確立主題和任務：這次要想的題目是什麼、要產出什麼、照什麼順序輪帽、哪些帽子交給 subagent。
2. **白帽盤點事實** — 列出已知的資訊、資料、實際案例，只放客觀內容，同時標出還缺什麼資訊。輸入：任務說明。
3. **綠帽發散** — 不設限地拋點子：怎麼做更好？有沒有替代方案？只產出、不評價，先把量衝起來。輸入：任務說明＋白帽的事實。
4. **黃帽找價值** — 對綠帽的每個主要選項問：優勢是什麼？如何做才能實現？輸入：任務說明＋選項清單。與黑、紅兩帽平行執行、互不相見。
5. **黑帽挑風險** — 對同一批選項問：可能的風險是什麼？外部有什麼阻力、競爭或替代選項？指出具體弱點，不是全盤否定。輸入同黃帽；因為看不到黃帽的樂觀評估，批判不會被帶風向。
6. **紅帽問直覺** — 誠實寫下對各選項的直覺好惡，不必給理由。輸入同黃帽；因為沒讀過黃、黑的分析，直覺不受污染。
7. **藍帽收斂（主 agent）** — 回收全部產出：整理、分類、結合點子，決定方案或下一步。發現某個觀點太薄，就再派那頂帽子補一輪。

不支援 subagent 的環境，退回單人輪流戴帽：照同樣順序一次只用一種觀點思考，戴哪頂帽就只說那頂帽的話。

### 常見錯誤

- 混帽：戴著綠帽卻一直挑毛病。一次只能用一種觀點，黑帽的話留到黑帽再說。
- 跳過不順手的帽子：樂觀的人跳過黑帽、務實的人跳過綠帽。六頂全戴，這正是跳脫自己立場的方法。
- 紅帽硬編理由：紅帽只要直覺，不需要辯護；逼理由會把情緒包裝成假邏輯。
- 黑帽變成唱衰大會：黑帽要的是具體風險與弱點，不是情緒性否定——那是紅帽的事。
- 綠帽自我審查：發散階段就先過濾「不可行」的點子。量不夠，就淬煉不出新奇的看法。
- 沒有藍帽收斂：六頂戴完點子散一地，沒有整理、分類、結合成方案，發散就白做了。

### 自檢清單

做完六頂思考帽後逐項檢查，任何一項不成立就回去補：

- [ ] 六頂帽子全部輪過，沒有跳過任何一頂
- [ ] 用 subagent 分飾時，各帽只拿到自己需要的輸入，互相看不到彼此的發言
- [ ] 每頂帽子的產出只屬於該帽觀點，沒有混帽
- [ ] 白帽列的是事實與案例，並標出缺少的資訊
- [ ] 綠帽沒有自我審查，點子有先衝量再說
- [ ] 黃帽對主要選項都講得出優勢與實現路徑
- [ ] 黑帽指出的是具體風險與弱點，不是空泛唱衰
- [ ] 紅帽誠實寫下好惡，沒有硬編理由
- [ ] 藍帽有開場定題、收場彙整，並產出決定或下一步

## 腳本圖：視覺化模糊想法，便於共享、討論

腳本圖（storyboard）可以具體化模糊的想法，幫助理清思緒，也便於和他人分享念頭、改善點子。起初多是動畫和電影公司採用；企業則應用於理解顧客旅程，以四格漫畫畫出「有問題的現狀」「解決問題的過程」「問題解決後的狀況」。在發散與收斂中偏收斂：把無形的想法化為有形。一句話：**視覺化模糊想法，便於共享、討論。**

想法還模糊、講不清楚，或要讓別人快速看懂一個構想時都適用：說明一個方案的使用經驗、設計任何「有人會經歷的流程」、鋪一條敘事線。不一定要真的畫圖，文字分格（每格編號＋場景描述）就有效果；環境能產圖再配圖。

### 操作步驟

1. **定主角與想法** — 寫下要具體化的想法，以及誰會經歷它（使用者、顧客、讀者、任何當事人）。腳本圖全程用主角的視角。
2. **切故事骨架** — 依「有問題的現狀 → 解決問題的過程 → 問題解決後的狀況」分配格子。四格起手（1 格現狀、2 格過程、1 格結局），想法複雜可以加格，但每格只演一個場景。
3. **逐格寫場景** — 每格一句話：主角在哪、做什麼、感覺什麼。要具體到「畫得出來」；畫不出來代表還太抽象，繼續具體化。
4. **檢查故事線** — 從第一格唸到最後一格：主角的狀態有沒有真的改變？格與格的轉場合理嗎（主角為什麼會走到下一格）？唸不順的斷點就是想法的缺口。
5. **分享與改善** — 拿給別人看，收集「哪一格不合理、哪一格最有感」，回頭改點子。Agent 情境可另開一個 subagent 以主角身分重走每一格，挑出不合理處。

### 對照範例：Airbnb 客戶體驗

| 格 | 場景 | 段落 |
| --- | --- | --- |
| 1 | 想找安心、價格實惠，又在地的旅遊住宿 | 有問題的現狀 |
| 2 | 在 Airbnb 找到符合預算、誠信、又特別的住宿 | 解決問題的過程 |
| 3 | 和房東一起吃當地美食、遊覽私房景點 | 解決問題的過程 |
| 4 | 開心返家，保持聯繫，成為朋友 | 問題解決後的狀況 |

四格全程是顧客視角；最後一格把價值主張演出來——不只是訂到房，而是交到朋友。

### 常見錯誤

- 格子裡寫抽象概念（「提升體驗」）而不是場景。畫不出來的，就還不是腳本圖。
- 只畫解決方案，沒畫「有問題的現狀」。沒有 before，看不出方案的價值。
- 停在「用了方案」，沒畫「問題解決後的狀況」。最後一格才是價值主張所在。
- 視角跑掉：畫成提供者視角的功能展示，主角不見了。
- 格與格跳太遠，轉場靠讀者腦補。
- 畫完就收，沒拿去分享、收回饋、改點子。便於共享討論正是它的目的。

### 自檢清單

做完腳本圖後逐項檢查，任何一項不成立就回去補：

- [ ] 主角明確，全程是主角視角
- [ ] 三段俱全：有問題的現狀、解決的過程、解決後的狀況
- [ ] 每一格都是具體場景，具體到畫得出來
- [ ] 從頭唸到尾，主角的狀態有真的改變
- [ ] 格與格的轉場合理，沒有靠腦補的跳躍
- [ ] 有分享或用主角視角重走一遍，並根據回饋改過點子

## 心智圖：窮盡選項，拓寬思考廣度

相較線性思考，英國心理學家東尼・博贊（Tony Buzan）提出的心智圖法，將思維從中心點發想，向外擴散主幹、枝幹：主題置於中央，畫出第一層主幹，再往外聯想枝幹，愈具體愈好，逐步理出架構和關聯性，最後選擇優先執行的項目。在發散與收斂中偏發散：先把選項窮盡、把廣度拓開。一句話：**窮盡選項，拓寬思考廣度。**

題目剛起步、還不知道有哪些面向時最適用：盤點一個主題的全貌、列大綱、規劃一件多面向的事、找出被忽略的選項。原方法靠顏色與圖像強化視覺；純文字環境用縮排樹狀呈現分支就有同樣效果，能產圖的環境再畫成圖。

### 操作步驟

1. **置中主題** — 把要發想的主題寫在中央，一句話講清楚這次要窮盡的是什麼。
2. **拉第一層主幹** — 決定 4–7 條主幹當大分類。起手式可用人事時地物、5W1H、SWOT 等現成框架，或依主題自訂（開咖啡店：行銷、產品、管理、空間規畫）。主幹的切分用 MECE 把關：不重疊、重要面向不遺漏。
3. **向外長枝幹** — 對每條主幹自由聯想往外展開，一層比一層具體（產品 → 咖啡 → 美式、義式）。每個節點用短詞，不寫長句。這一步只衝廣度，不評價、不篩選。
4. **理出架構與關聯** — 全圖掃一遍：放錯家的節點搬家、重複的合併、跨分支的關聯標出來（行銷的促銷支出，會出現在管理的變動成本裡）。
5. **選擇優先執行項目** — 從全圖挑出這次要先做的少數項目，說明為什麼是它們（影響大、急迫、可行）。心智圖負責拓寬，收斂靠這一步收口。

### 對照範例：開一間咖啡店

```
開一間咖啡店
├─ 行銷：價格／促銷（買一送一、異業結盟聯名合作）／明星產品（手工蛋糕、冠軍咖啡豆）
├─ 產品：果茶（當季水果）／咖啡（美式、義式）／點心（蛋糕、餅乾）
├─ 管理：成本（固定：水電、租金、人事；變動：原料、行銷）／人員（內外場、教育訓練）
└─ 空間規畫：裝潢風格（工業、鄉村、甜美）／用途（聊天－採光、工作－插座）／座位數（10、20、30 人）
```

四條主幹互不重疊、涵蓋開店的主要面向；愈外層愈具體；跨分支關聯（行銷 ↔ 變動成本）掃圖時標出來，常是最有價值的洞察。

### 常見錯誤

- 中央主題太模糊，發散沒有錨點，枝幹長歪。
- 第一層主幹想到什麼寫什麼，沒用框架或 MECE 把關，重要面向整條漏掉。
- 枝幹愈長愈抽象。方向反了，愈外層要愈具體。
- 節點寫成長句，圖變成段落的樹狀排版，聯想的節奏斷掉。
- 太早篩選：枝幹階段就先審查「這不可行」，廣度衝不開。
- 只發散不收斂：畫完一大張，沒理架構、沒挑優先項目，廣度沒有轉成行動。

### 自檢清單

做完心智圖後逐項檢查，任何一項不成立就回去補：

- [ ] 中央主題一句話，清楚這次要窮盡什麼
- [ ] 第一層主幹 4–7 條，用框架或 MECE 檢查過不重不漏
- [ ] 每條主幹都往外長了枝幹，愈外層愈具體
- [ ] 節點都是短詞，不是長句
- [ ] 發散階段沒有先篩選，廣度有衝開
- [ ] 掃過全圖：錯位的搬家、重複的合併、跨分支關聯有標出
- [ ] 有挑出優先執行的項目，並說得出為什麼是它們
