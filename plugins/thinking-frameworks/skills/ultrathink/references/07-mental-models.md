# 思維模型

## 主題定位

這一組是跨領域通用的思維模型（mental models）。查理・蒙格（Charlie Munger）主張在腦中建立「思維模型的格柵」（latticework of mental models）：把來自不同學科的基本模型交織起來用，避免「手裡只有錘子，看什麼都像釘子」的單一視角偏誤。

與前面的主題不同，這裡的模型不是流程而是思考紀律，任何任務的任何階段都能調用。在本 skill 的序列中它排最後，兼作總體檢：把前面各框架得出的結論拿來過這幾道紀律，確認想得夠本質、夠周延、夠誠實、夠簡潔。

## 第一性原理：拆解到最基礎的事實，從零構建

亞里斯多德把第一性原理定義為「認識事物的最初基礎」——不能再從更基本的東西推導出來的命題。伊隆・馬斯克（Elon Musk）讓它重回大眾視野：「我們過日子多半靠類比推理——照著別人的做法微調。第一性原理是把事情煮沸到最基本的事實，再從那裡往上推理。」一句話：**回歸事物的本質，將問題拆解為最基礎的真實事實，再從零構建解決方案，而不依賴類比。**

適用時機：被「一直都是這樣做」卡住、成本或做法看起來不合理卻說不出為什麼、要做沒有前例的東西、類比推理給不出突破時。它是昂貴的工具——關鍵問題才值得動用，日常決策用類比就好。

### 操作步驟

1. **寫下現行認知** — 要解的問題是什麼？目前的預設方案或「大家都這樣做」的版本長什麼樣？
2. **逐條拆出前提並質疑** — 把現行認知拆成一條條前提，逐條問：這是可驗證的事實，還是慣例、類比、道聽塗說？憑什麼成立？
3. **煮沸到基礎事實** — 留下不能再拆、可驗證的元素：物理或邏輯限制、明確的需求、查證過的數據、定義本身。每條都要通過「這是事實，不是沿襲」的檢查。
4. **從零重新構建** — 暫時禁用「別人怎麼做」，只從基礎事實往上推：如果從來沒有人做過這件事，根據這些事實，最合理的做法是什麼？
5. **對照與落地** — 把重建方案跟現行做法比：差在哪、為什麼差、哪些差異值得執行。重建完可以再借既有做法補實作細節，但方向由事實決定，不由慣例決定。

### 對照範例：電池成本

類比推理：「電池組一直都是每 kWh 約 600 美元，很貴，未來也會很貴。」第一性原理：電池由什麼構成？鈷、鎳、鋁、碳、分離膜、外殼——把這些材料在市場上買齊，約每 kWh 80 美元。可見貴的不是本質成本，而是製造與供應鏈的結構。重建結論：改變製造方式，成本可以大幅下降——後來電池價格的長期走勢證實了這個推理。

### 常見錯誤

- 把慣例當事實：「業界標準」「一直都這樣」混進基礎事實層。
- 拆得不夠深：停在「成本就是這麼高」這種還能再拆的中間層。
- 全盤否定類比：類比是便宜高效的捷徑，第一性原理是昂貴的工具，用錯場合是浪費，不是深刻。
- 重建時偷渡舊方案：號稱從零構建，其實把原方案換句話說。
- 基礎事實沒驗證：拆出來的「事實」本身是道聽塗說，整個重建蓋在沙上。

### 自檢清單

做完第一性原理後逐項檢查，任何一項不成立就回去補：

- [ ] 現行做法與它的前提有逐條攤開
- [ ] 每個前提都問過「事實還是慣例」，憑什麼成立
- [ ] 基礎事實層每一條都可驗證，沒有混入類比與沿襲
- [ ] 重建只從基礎事實出發，沒有偷渡舊方案
- [ ] 重建方案與現行做法的差異與理由說得清楚
- [ ] 有評估過這題值不值得動用第一性原理

## 反向思考法：先想怎麼避免失敗，再想怎麼成功

德國數學家雅可比（Carl Jacobi）解難題的心法是「man muss immer umkehren」——反過來想，總是反過來想。查理・蒙格把它變成招牌思維模型：「我只想知道我將來會死在哪裡，這樣我就永遠不去那裡。」他的總結更直白：像我們這樣的人能拿到長期優勢，靠的是持續不做蠢事，而不是追求非常聰明。很多問題正著解很難，反著解就清楚。一句話：**與其思考如何成功，不如先思考如何避免失敗，將問題反過來檢視。**

適用時機：規劃任何重要行動之前、方案看起來一路順利的時候、盤點風險、檢查盲點。制度化的版本是「事前驗屍」（pre-mortem，Gary Klein）：開工前假設「這件事一年後已經失敗了」，回推所有可能的死因。

### 操作步驟

1. **定目標** — 寫下想達成的結果。
2. **反轉提問** — 問「怎樣做會讓這件事徹底失敗、變得最糟？」把搞砸的方式全部列出來：會毀掉它的行為、遺漏、條件、外部事件，愈具體愈好，包含「什麼都不做」這種失敗法。
3. **失敗清單轉禁區** — 每條失敗路徑轉寫成「絕不做 X」或「必須確保 Y」，形成避錯清單。
4. **正反合一** — 回到正向方案，逐條對照避錯清單：現在的計畫有沒有正踩在某條失敗路徑上？先消除愚蠢，再追求聰明。
5. **設反向檢查點** — 在執行的關鍵節點重跑一次：「接下來要做的事，是不是失敗清單上的哪一條？」

### 對照範例：一篇想被讀完的文章

正向問「怎麼寫出好文章」很發散；反轉問「怎樣保證沒人讀完」立刻具體：開頭三行抓不到重點、通篇沒有讀者在乎的問題、段落之間沒有邏輯、標題與內文不符。逐條反轉成禁區，回頭對照草稿排雷——每一條都可執行、可檢查。

### 常見錯誤

- 失敗列得太泛：「做不好」「品質差」避無可避。失敗要具體到能反轉成行動。
- 反完不回正：失敗清單沒有轉成禁區與檢查點，只是一張焦慮清單。
- 把反向當悲觀：這是排雷不是唱衰，排完雷正向計畫走得更敢。
- 只在開頭做一次：執行中的新決定沒有再過反向檢查。
- 漏掉不作為：有時最大的失敗路徑是什麼都不做，或動得太晚。

### 自檢清單

做完反向思考後逐項檢查，任何一項不成立就回去補：

- [ ] 目標明確，反轉提問問得具體
- [ ] 失敗路徑夠多夠具體，包含不作為與外部事件
- [ ] 每條失敗路徑都轉成禁區或必要保障
- [ ] 正向方案逐條對照過避錯清單
- [ ] 執行的關鍵節點安排了反向檢查
- [ ] 排完雷有回到正向行動，沒有停在悲觀

## 二階思考：多問一句「然後呢」，推演連鎖反應

霍華・馬克斯（Howard Marks）在《投資最重要的事》提出「第二層思考」：第一層思考說「這是好公司，買」；第二層思考說「這是好公司，但所有人都覺得它好，價格早已反映，該賣」。第一層思考人人會，所以不可能帶來超越平均的判斷品質，優勢來自往下多推幾層。核心提問只有一句：「然後呢？（And then what?）」一句話：**評估決策時，不僅看當下的結果（第一階），更推演該決定所引發的後續連鎖反應（第二、三階）。**

適用時機：任何有後續效應的決定——規則、誘因設計、架構選擇、對外承諾、政策；特別是「看起來明顯該做」的決定，因為明顯的好處人人看得到，藏在第二階的代價沒人看。

### 操作步驟

1. **寫下決策與一階結果** — 這個決定直接造成什麼？大多數思考停在這裡。
2. **連問「然後呢」** — 對每個一階結果追問：接下來會發生什麼？至少推到第二、三階。
3. **分時間尺度** — 短期、中期、長期各是什麼樣子？很多決定短期爽、長期痛；反過來，一階痛、二三階大好的決定常被低估。
4. **推演各方反應** — 有其他參與者時（使用者、對手、團隊、上下游）：他們會怎麼因應你的行動？誘因會不會被鑽？反應會不會抵銷甚至反轉一階的好處？
5. **綜合取捨** — 一階好、二階壞的要慎防；一階痛、長期好的重新評估。把推演結果餵回決策工具（決策矩陣、力場分析）。

### 對照範例：眼鏡蛇效應

殖民時期的德里為了滅蛇而懸賞蛇屍。一階：蛇被抓，數量下降。二階：有人發現養蛇領賞更有效率，開始養蛇。三階：政府察覺後取消懸賞，養殖的蛇被放生——蛇比政策實施前更多。誘因設計只推了一階，被第二階反噬。

### 常見錯誤

- 停在一階：最普遍的失敗。「這樣做有效」——然後呢？
- 推演變成空想：二、三階要基於「誰、為什麼、會怎麼反應」的具體機制，不是自由聯想。
- 只推單邊：只推壞處變成杞人憂天，只推好處變成自我說服。兩邊都要推。
- 忽略時間尺度：短期與長期效應可能相反，混在一起看會誤判。
- 推演癱瘓：推到第五階不敢動。重點是找到「會反轉結論的那一層」，通常在二、三階。

### 自檢清單

做完二階思考後逐項檢查，任何一項不成立就回去補：

- [ ] 一階結果寫清楚
- [ ] 每個一階結果都問過「然後呢」，至少推到二階
- [ ] 短中長期分開看過，檢查有沒有先甘後苦或先苦後甘
- [ ] 各參與者的反應推演過，特別是誘因會不會被鑽
- [ ] 好與壞兩個方向都推了
- [ ] 推演結果有回饋到決策，不是推完就算

## 能力圈：知道邊界在哪，比圈子多大更重要

華倫・巴菲特（Warren Buffett）在 1996 年致股東信寫道：「你不需要成為每家公司的專家，只需要能評估你能力圈之內的公司。圈子的大小不太重要，但知道它的邊界，至關重要。」查理・蒙格的做法更直白：「我們有三個籃子：進、出、太難。大多數東西都被丟進『太難』那籃。」一句話：**誠實評估自己真正了解的領域，在邊界內做出精準決策，並在邊界外保持謹慎或尋求專家協助。**

適用時機：每一次要下判斷之前，尤其是判斷「看起來很簡單」的時候。對 agent 而言這就是幻覺防治：知識邊界內果斷作答，邊界外查證、標示不確定或求助，絕不用流暢的語言蓋住不懂。

### 操作步驟

1. **畫圈** — 列出當前任務涉及的知識領域，逐一誠實分進三籃：真懂（能解釋原理、能預測結果、知道例外與反例）、略懂（用過聽過但講不出所以然）、不懂。
2. **判斷任務落點** — 這次的關鍵判斷落在圈內、圈邊，還是圈外？
3. **圈內果斷** — 真懂的範圍內給明確判斷，不無謂含糊。假謙虛也是一種失職。
4. **圈外誠實** — 落在圈邊或圈外的：查證、找專家或工具協助，或明說「這是推測，錯了會怎樣」。學蒙格把題目丟進「太難」籃，也是合法選項。
5. **測圈與擴圈** — 定期驗證：我以為懂的，做出的預測有沒有中？錯了就把圈縮回來。擴圈靠刻意學習加回饋，不靠自我感覺。

「真懂」的三個測試：能不能用簡單的話解釋給外行聽？能不能在這個領域做出會命中的預測？知不知道它的例外、反例與失效條件？

### 對照範例：躲過的泡沫

1990 年代末科技股狂飆，巴菲特因為「看不懂這些公司十年後的競爭力」而不買，一度被嘲笑跟不上時代；dot-com 泡沫破裂後，這個「不動」成了教科書案例。重點不是科技股好壞，而是他清楚那在自己的圈外——圈外的機會再誘人，也是別人的機會。

### 常見錯誤

- 略懂當真懂：最危險的一格。知道名詞不等於懂原理，用過不等於能預測。
- 假謙虛不決策：圈內也不敢下判斷，能力圈變成免責藉口。
- 圈外硬撐：不查證不求助，用自信的語氣和流暢的行文蓋住不懂。
- 把圈當靜態：領域會變，昨天的真懂會過期，不重測就是拿舊地圖走新路。
- 比圈子大小：重點從來不是圈多大，是邊界清不清楚。

### 自檢清單

做完能力圈檢視後逐項檢查，任何一項不成立就回去補：

- [ ] 任務涉及的領域列出來，逐一分了真懂、略懂、不懂三籃
- [ ] 關鍵判斷的落點（圈內、圈邊、圈外）判斷過
- [ ] 圈內判斷明確果斷，沒有無謂含糊
- [ ] 圈外部分有查證、求助，或明確標示為推測並說明風險
- [ ] 沒有用自信語氣包裝圈外的猜測
- [ ] 「真懂」用三個測試檢過，不是自我感覺
