# 權衡得失

## 主題定位

《精進思維》指出，人們面臨抉擇時總是傾向滿意原則（satisficing）而非最優原則。滿意原則是從自己最熟悉的待選選項逐一考察，發現能滿足過往經驗標準的選項，就毫不猶豫採用；最優原則採全域視角，系統化盤點、比較所有可見選項，雖然費上更多工夫，最後的決策品質也愈高。

套用到任何任務上：方向、方案、寫法、架構的選擇，都值得從「第一個看起來可以的」升級成「盤點過所有可見選項再挑」。這個主題的工具把選擇變成可檢驗的計算與比較，決定不憑直覺。

## 決策矩陣：加權計算優缺點，決定不憑直覺

每項提案與點子都各有優缺點，是做決定最大的痛點。決策矩陣為提案的每個面向量化分析：先羅列最關鍵的幾項標準並給予權重，接著計算每個選項的得分，採用總分最高的提案。常見的分析指標有可行性、未來性、創新性、利益性等；想找各指標比較均衡的提案，可以採用雷達圖。一句話：**加權計算優缺點，決定不憑直覺。**

任何多選項的決定都適用：方案取捨、工具選型、方向定奪、資源分配。前面工具的發散產出（心智圖的選項、六帽綠帽的點子）正好餵進來比較。

### 操作步驟

1. **列出選項** — 把要比較的選項列齊。滿意原則的陷阱就是只看熟悉的選項——先系統化盤點所有可見選項，再進矩陣。
2. **定關鍵標準** — 羅列最關鍵的幾項評估標準（可行性、未來性、創新性、利益性、成本等，依任務自訂）。數量抓少而關鍵，標準之間不重疊——兩個高度相關的標準等於偷偷加權重。
3. **給權重** — 依重要程度給權重（如 ×3、×2、×1）。權重反映這次決策在乎什麼，每個權重都要說得出理由。
4. **逐項評分** — 每個選項在每個標準上打分。評分前先寫下基準（幾分代表什麼）；給分方向一致，愈好愈高分——「愈低愈好」的指標（如成本）反向給分。分數要有依據，不是印象分。
5. **加權加總，採用最高分** — 每格「分數 × 權重」加總，採用總分最高的提案。總分接近時做敏感度檢查：權重或給分微調會不會翻盤？會的話代表結論不穩，回頭補強依據再定。
6. **要均衡就看雷達圖** — 想找各指標比較均衡的選項，把各選項分數畫成雷達圖：形狀勻稱＝均衡，總分高但形狀尖銳＝偏科。總分與均衡度並看，依任務性質取捨。

### 對照範例：兩個提案

| | 可行性 ×3 | 未來性 ×2 | 創新性 ×1 | 利益性 ×3 | 成本 ×2 | 合計 |
| --- | --- | --- | --- | --- | --- | --- |
| 提案 1 | 7 | 4 | 5 | 8 | 2 | **62** |
| 提案 2 | 5 | 2 | 7 | 4 | 5 | **48** |

提案 1 總分勝出（7×3＋4×2＋5×1＋8×3＋2×2＝62），採用提案 1。但注意它的成本分只有 2，弱項明顯——若這次任務成本是硬限制，權重就不該只有 ×2。畫成雷達圖更清楚：提案 1 在可行性、利益性突出但成本凹陷，提案 2 形狀較勻稱；要衝刺選前者，要穩定選後者。

### 常見錯誤

- 只把熟悉的選項放進矩陣：滿意原則的陷阱。矩陣再嚴謹，選項清單缺漏就是假最優。
- 標準互相重疊：等於偷偷重複加權，總分被相關的標準灌水。
- 權重憑感覺或全部一樣：權重是這次決策價值觀的宣告，說不出理由就會扭曲結論。
- 給分方向不一致：成本這種愈低愈好的指標忘了反向給分，總分失真。
- 印象分：分數沒有依據。先寫評分基準再打分。
- 把矩陣當橡皮圖章：總分輸了卻偷改分數遷就直覺。對結果不服，回頭誠實改權重並說明理由，不要動分數。
- 只看總分不看形狀：需要全面穩定的任務，總分最高的偏科選項反而危險。

### 自檢清單

做完決策矩陣後逐項檢查，任何一項不成立就回去補：

- [ ] 選項有系統化盤點過，不是只列熟悉的幾個
- [ ] 標準少而關鍵、彼此不重疊，涵蓋這次決策在乎的面向
- [ ] 每個權重都說得出理由
- [ ] 給分方向一致，愈低愈好的指標有反向給分
- [ ] 評分基準先寫好，每個分數有依據
- [ ] 加權計算驗算過，總分接近時做過敏感度檢查
- [ ] 需要均衡的任務有用雷達圖看過形狀
- [ ] 採用的是計算結果，沒有事後偷改分數遷就直覺

## PMI 表：舉出優劣點外的有趣點，評估更全面

有時候，當時認為無關緊要的因素，之後卻發現重要性。心理學家愛德華・狄波諾（Edward de Bono）提出 PMI 分析法，建議決策者除了優點（Plus）、缺點（Minus）外，還要考慮有趣點（Interesting）：日後有其他變數才能判斷優劣的因素，也需納入討論，以免因為顯而易見的優缺點，忽略潛在的影響因素。一句話：**舉出優劣點外的有趣點，評估更全面。**

任何單一選項的評估都適用：該不該做某個決定、採不採用某個方案。跟六頂思考帽同一位作者：P、M 近似黃帽與黑帽的視角，I（有趣點）是這個工具獨有的第三眼。跟決策矩陣併用時，PMI 先把每個選項的因素找全（尤其潛在因素），矩陣再量化比較。

### 操作步驟

1. **定題目** — 把要評估的決定寫成一句問句：「該買新建案的房子嗎？」「該採用這個方案嗎？」
2. **P：列優點** — 顯而易見的好處逐條列，每條具體到可檢驗。
3. **M：列缺點** — 顯而易見的壞處逐條列，同樣具體。
4. **I：列有趣點** — 關鍵步驟。掃一遍「注意到了但說不上好壞」的因素：現在無法判斷優劣、日後有其他變數才能判斷的，全部列進來，不要因為看似無關緊要就丟掉。對每個有趣點問：什麼變數會讓它變優點？什麼變數會讓它變缺點？
5. **整體評估** — 三欄一起看：優缺點給當下判斷，有趣點給潛在變化。下結論時標出哪些有趣點值得追蹤或查證，把它們變成後續的驗證項。

### 對照範例：該買新建案的房子嗎？

| 欄 | 內容 |
| --- | --- |
| ＋ 優點 | 離車站近、裝潢時髦、廚房設備齊全、親友住附近 |
| － 缺點 | 售價比預期高一點、總坪數稍小、外牆顏色黯淡 |
| ！ 有趣點 | 緊鄰大片空地、鄰居愛聊天、附近有許多貓 |

有趣點正是「日後有變數才能判斷」的示範：緊鄰大片空地——蓋成公園是優點、蓋起高樓擋視野是缺點，取決於日後開發，買前值得查空地的規畫用途；鄰居愛聊天——喜歡熱絡是優點、重隱私是缺點，取決於自己的偏好。

### 常見錯誤

- 只做 P 和 M：那只是優缺點清單。這個工具的價值在 I，漏了它等於沒用 PMI。
- 有趣點被硬塞進優缺點：還不能判斷好壞的因素被迫選邊，潛在因素被過早定性。
- 「無關緊要」直接丟掉：當時認為無關緊要的因素，之後卻發現重要性。說不上好壞不等於不重要。
- 有趣點列了不處理：沒問「什麼變數會讓它翻面」、沒排追蹤查證，列心安的。
- 優缺點寫成感想：每條要具體可檢驗——「離車站近」「售價比預期高一點」都查證得了。
- 三欄失衡不自覺：滿滿優點、零星缺點，可能是立場先行。回頭刻意補掃弱的那兩欄。

### 自檢清單

做完 PMI 表後逐項檢查，任何一項不成立就回去補：

- [ ] 題目寫成一句明確的問句
- [ ] P、M 每條都具體可檢驗，不是感想
- [ ] I 欄認真掃過，說不上好壞的因素都列了，沒有因為看似無關緊要而丟掉
- [ ] 每個有趣點都問過：什麼變數讓它變優點、什麼變數讓它變缺點
- [ ] 值得追蹤或查證的有趣點有標出來，成為後續驗證項
- [ ] 三欄份量檢查過，沒有立場先行的失衡
- [ ] 結論同時反映當下優缺點與潛在影響因素

## 力場分析：釐清選項的拉力與推力，評估更客觀

轉換跑道、移居海外等重大轉折時刻，做與不做的背後都各有充分的原因。心理學家庫爾特・勒溫（Kurt Lewin）提出力場分析：描述現狀與期望狀態間的差距，把驅動改變的推力（驅動力）與制約不變的拉力（制約力）用衝撞現狀的箭頭表示，以箭頭長短粗細標出個別力量的強度，思考該往哪個方向趨近才符合理想。一句話：**釐清選項的拉力與推力，評估更客觀。**

任何「做或不做某個改變」的評估都適用：轉向、改版、遷移、重構、退場。跟同主題工具的分工：PMI 找全單一選項的因素，力場分析把因素變成有方向、有強度的力，需根解損對變更提案做四關成立性裁決；矩陣則比較多個選項。純文字環境用表格＋強／中／弱（或 1–5）標強度，效果等同箭頭。

### 操作步驟

1. **寫下現狀與期望狀態** — 描述現在的處境與想趨近的理想狀態，兩者的差距就是要評估的改變。改變方向不明確，力就無從分邊。
2. **列驅動力（推力）** — 驅動改變發生的因素，逐條列出。
3. **列制約力（拉力）** — 制約現狀不變、阻擋改變的因素，逐條列出。
4. **標強度** — 為每股力量標強度（強／中／弱），依據是影響多大、多確定，不是憑感覺。同一因素不拆成多條灌水。
5. **判讀並找可動的力** — 比較兩側整體強度，思考該往哪個方向趨近。不只比大小：改變的路徑有兩條——增強驅動力，或削弱制約力；逐條問「這股力可變嗎？怎麼變？」行動打在可變的力上，而且削弱制約力常比硬加推力有效。

### 對照範例：是否提早退休？

| 驅動力（推向提早退休） | 強度 | 制約力（維持在職） | 強度 |
| --- | --- | --- | --- |
| 身體不如以往 | 強 | 家庭主要經濟支柱 | 強 |
| 已儲蓄足夠退休基金 | 中 | 沒什麼休閒嗜好 | 中 |
| 沒有升遷空間 | 強 | 公司未來獲利狀況樂觀 | 中 |
| 想多陪子女 | 弱 | 多數朋友仍在職場上 | 中 |

兩側都有強力，硬比總和難分高下，改看可變性：制約力裡「沒什麼休閒嗜好」可以培養、「家庭經濟支柱」可用漸進退場緩解；驅動力裡「身體不如以往」只會隨時間增強。往期望狀態趨近的路徑因此清楚：先削弱可變的制約力，而不是等推力自己變大。

### 常見錯誤

- 沒定義期望狀態：只列好壞因素，變成換皮的優缺點表。力場分析的錨是「現狀 → 期望狀態」的改變方向。
- 力量不標強度：每條等重。箭頭的長短粗細才是這個工具的獨有資訊。
- 強度憑感覺：標了強弱說不出依據。
- 只算總和就定案：兩邊抵銷式的算術。真正的價值在逐條找可動的力。
- 同一因素拆成多條：一股力寫三遍，白白加重某一側。
- 忽略可變性：定值的力（朋友都在職場）和可變的力（嗜好可以培養）混在一起看，行動打錯地方。

### 自檢清單

做完力場分析後逐項檢查，任何一項不成立就回去補：

- [ ] 現狀與期望狀態都寫清楚，改變方向明確
- [ ] 驅動力與制約力兩側都認真列過，沒有立場先行
- [ ] 每股力量都標了強度，且說得出依據
- [ ] 沒有同一因素拆多條灌水
- [ ] 逐條問過：這股力可變嗎？怎麼增強或削弱？
- [ ] 有區分可變與不可變的力，行動打在可變的力上
- [ ] 結論說得出該往哪個方向趨近、憑什麼

## 需根解損：四關逐一舉證，裁決變更提案是否成立

政策性辯論用四大爭點檢驗一個改變現狀的提案：需要性、根屬性、解決力、損益比。正方要讓提案成立，四關都得舉證過關；任何一關倒下，提案就不成立。套用到任務上，任何「我打算這樣做」都可以重述成一個變更提案來檢——功能變更、流程改動、重構、debug 的修法、文案的切角都適用。一句話：**四關逐一舉證，提案不成立就不動手。**

| 關卡 | 檢驗問題 |
| --- | --- |
| 需（需要性） | 現狀是否有問題，或有可獲得的利益？有沒有改變的理由？ |
| 根（根屬性） | 問題是否根植於現行制度或做法本身？提案是否對症下藥？ |
| 解（解決力） | 提案實施後，真的能有效解決問題、實現利益嗎？ |
| 損（損益比） | 提案帶來的利益，是否大於衍生成本與負面影響？ |

跟同主題工具的分工：PMI 找全單一選項的因素，力場分析看改變的推拉力，需根解損做成立性總裁決；決策矩陣比較的是「改成哪個」，需根解損檢驗的是「該不該改」。

原方法是辯論賽制，四個爭點由不同辯士分頭攻防。Agent 使用時，優先用 subagent 還原這個結構：四關各開一個獨立 context 的 subagent，每個只拿到現狀描述、提案內容與自己那一關的檢驗問題，看不到其他關的結論，立場才不會互相帶風向。主 agent 負責統整裁決。環境不支援 subagent 時，才退回單人依序分飾四關。

### 操作步驟

1. **寫下現狀與提案** — 一段現狀描述（含已知事實與依據）、一段提案內容（要改什麼、怎麼改）。這是四個 subagent 共用的唯一輸入，措辭保持中立，不夾帶支持或反對的立場。
2. **平行開四關 subagent** — 每個 subagent 的提示只含：現狀、提案、該關的檢驗問題，並要求逐點舉證、標明假設、給出該關成立與否的結論。四關互不相見。
   - **需**：找現狀的實際問題或未實現的利益，每一條附依據；找不到就誠實回報「無改變理由」，不硬湊。
   - **根**：追問題的成因是否出在現行制度或做法本身。若問題來自執行不力、外部因素或一次性事件，改做法就不對症；同時檢查提案打的是不是那個成因。
   - **解**：推演提案實施後的因果鏈，逐環檢查是否成立、依賴了哪些未驗證的假設、有沒有繞過方案的路徑——能被規避的方案，解決力存疑。
   - **損**：盤點提案的衍生成本與負面影響（實施成本、轉換成本、新風險、副作用、機會成本），與預期利益比較。
3. **依序裁決（主 agent）** — 回收四關結論，照 需→根→解→損 的順序裁決。這是舉證鏈：前一關不成立，後面的結論只當參考，不能拿「損益比划算」救一個根本不需要或不對症的提案。
4. **給出裁決與修補方向** — 四關全過：提案成立，把各關標出的假設列為後續驗證項。任何一關倒下：指出倒在哪一關、為什麼，並提出修補方向（縮小提案、換打點、先補驗證），不是只說不行。

### 對照範例：該新增積分獎勵系統嗎？

現狀：產品月流失率 40%，高於同類產品。提案：新增積分獎勵系統，提高使用者留存誘因。

| 關卡 | 檢驗結果 | 裁決 |
| --- | --- | --- |
| 需 | 流失率 40% 有數據佐證，確有改變的理由 | 過 |
| 根 | 使用者訪談顯示流失主因是核心功能載入過慢，不是缺乏誘因——提案沒打中成因 | **倒** |
| 解 | （參考）就算做了，對載入慢造成的流失沒有解決力 | — |
| 損 | （參考）開發兩個月＋長期維運成本 | — |

裁決：倒在根屬性。解與損的結論只當參考，不回頭用「積分系統成本不高」翻案。修補方向：提案改成效能優化；積分系統留待流失主因排除後再評估。

### 常見錯誤

- 只檢解與損，跳過需與根：直接辯「方案好不好」，沒先確認「該不該改」。多數不成立的提案倒在前兩關。
- 四關看得到彼此的結論：解的樂觀推演帶壞損的成本盤點，需的問題清單引導根往制度面找——立場互相污染，四關變一言堂。
- 把「有問題」當成「需要這個提案」：需只證明改變的理由，不證明這個提案；對不對症是根的事，有沒有效是解的事。
- 根屬性只問「有沒有問題」不問「成因在哪」：執行面的問題被拿來當改制度的理由。
- 拿後面的關卡救前面：損益比再划算，也救不了不需要或不對症的提案。
- 裁決只說不行不給修補方向：這是動手前的思考工具，目的是把提案修對，不是純否決。
- 現狀與提案描述夾帶立場：輸入偏了，四關全偏。

### 自檢清單

做完需根解損後逐項檢查，任何一項不成立就回去補：

- [ ] 現狀與提案寫成中立描述，沒有夾帶支持或反對的措辭
- [ ] 四關各自獨立檢驗：subagent 互不相見，單人分飾時沒讓前一關的結論帶風向
- [ ] 需：改變的理由有依據，不是印象
- [ ] 根：問題成因追到了，且確認提案打的就是那個成因
- [ ] 解：因果鏈逐環檢過，依賴的假設有標出來
- [ ] 損：衍生成本與負面影響盤點過，有跟預期利益比較
- [ ] 裁決照 需→根→解→損 順序，沒有拿後面的關卡救前面
- [ ] 不成立時有指出倒在哪一關與修補方向
