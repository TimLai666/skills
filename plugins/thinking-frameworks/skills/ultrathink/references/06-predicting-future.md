# 預測未來

## 主題定位

《多模型思維》指出，模型可以清楚、有邏輯地解釋現象：經濟模型解釋價格變化和市占率，物理模型解釋物體下墜速度和飛行軌道。當數據愈來愈容易取得、資料也更精細，人們愈來愈常使用模型預測未來——社群和網路搜尋資料，也能用來預測消費趨勢和社會現象。

當結論取決於生命週期、採用擴散、時間變化或頭尾配置時，路由到本類別。選中後三個工具全部依序執行，推演趨勢往哪走、目前做法還能有效多久，以及何時該換打法。

## S 型曲線：你的產品壽命還剩多久？

產品的成長曲線通常為「S 型曲線」，分成三個階段：嬰兒期、擴張期（成長期）和成熟期，前兩期呈現指數成長；到了最後階段，產品銷量已經到達極限，必須再思考新的產品及策略，否則就會進到第四階段：衰退期。這四個階段合稱「產品生命週期」（product life cycle）。一句話：**判斷所在階段，抓準換打法的時機。**

不只產品：技術、方法、內容形式、通路，任何「會成長、會飽和」的東西都走 S 曲線。用兩個訊號就能對出階段：

| 階段 | 市場成長率 | 競爭者 | 曲線特徵 |
| --- | --- | --- | --- |
| 1 嬰兒期 | 高 | 少 | 起步緩慢，銷售額低，利潤低或為負 |
| 2 成長期（擴張期） | 高 | 增加 | 指數成長，利潤開始跟上 |
| 3 成熟期 | 低 | 多 | 銷量到達極限，銷售額與利潤見頂 |
| 4 衰退期 | 低 | 減少 | 銷售額與利潤下滑 |

### 操作步驟

1. **定觀察對象與指標** — 要判斷壽命的對象是什麼（產品、服務、技術、內容形式、方法），用什麼指標衡量它的成長（銷售額、使用量、聲量）。對象界定要準：「整個產業」和「單一產品」的階段可能完全不同。
2. **判斷所在階段** — 用「成長率高低 × 競爭者多寡與增減」對照上表定位階段，再用利潤（效益）曲線佐證：利潤落後於銷售額——嬰兒期低、成長期跟上、成熟期見頂、衰退期雙降。
3. **推估曲線位置與剩餘壽命** — 成長還在加速（指數段），還是開始放緩（接近極限）？極限大概在哪（市場天花板、飽和訊號）？
4. **依階段擬策略** — 嬰兒期驗證與卡位；成長期擴張搶份額；成熟期顧效率守成果，同時必須開始思考新的產品及策略——否則就會進入衰退期；衰退期收割、轉型或退場。
5. **佈局下一條曲線** — 成熟期就是啟動下一條 S 曲線的時機，不要等衰退才動。根據本次分析列出新曲線的候選方向；若後續需要擴充選項或驗證市場，再把創意發想或市場分析記為新的路由候選，不在本輪自動展開。

### 常見錯誤

- 把一時的成長當永恆：以為指數段會一直持續，成熟期還用成長期的打法。
- 階段誤判：只看銷售額絕對值。銷售額最高的時候往往已是成熟期，要看成長率與競爭者。
- 只看銷售不看利潤：利潤落後於銷售，走下坡的訊號常先出現在利潤上。
- 等衰退才找新品：成熟期就要思考新產品及策略，衰退期才動就晚了。
- 對象界定不清：拿產業的曲線判斷單一產品，或反過來，階段全對不上。
- 把 S 曲線當精準預言：它是階段判斷與時機工具，不是數值預測，要用實際數據滾動修正。

### 自檢清單

做完 S 型曲線分析後逐項檢查，任何一項不成立就回去補：

- [ ] 觀察對象與成長指標定義清楚，範圍界定正確
- [ ] 用成長率 × 競爭者兩個訊號判斷過階段，不是憑感覺
- [ ] 有對照利潤（效益）曲線佐證階段判斷
- [ ] 說得出目前在曲線哪一段、極限大概在哪
- [ ] 策略跟階段相符，沒有在成熟期用成長期打法
- [ ] 已在成熟期的話，新產品與策略的思考已經啟動
- [ ] 判斷有數據或可查證的訊號支撐，並保留滾動修正空間

## 鴻溝理論：產品能進入主流市場嗎？

近似掌上型電腦的 PDA 早期雖有許多愛好者，卻始終無法進入主流市場，這種現象稱為「跌入鴻溝」。矽谷管理諮詢家傑佛瑞・墨爾（Geoffrey Moore）提出：科技產品的普及分為早期市場與主流市場，兩者間有個難以跨越的鴻溝，不能在時間內跨越，就無法進入主流市場。兩個群眾可再細分成五個市場區隔，產品想成功，就得逐步攻克每個區隔。一句話：**判斷能否從早期市場跨進主流市場。**

任何要被一群人採用的東西都適用：產品、服務、內容、方法、把新工具或新流程推進團隊。它接在 S 曲線前段：跨得過鴻溝，才進得了高速成長段。

| 區隔 | 市場 | 在意什麼 |
| --- | --- | --- |
| 創新者 | 早期市場 | 新事物本身，搶先嘗試 |
| 早期採用者 | 早期市場 | 新事物帶來的先發優勢，願意冒險 |
| ——鴻溝—— | | 兩側動機本質不同，難以跨越 |
| 早期大眾 | 主流市場 | 完整方案和便利，要看得到實績 |
| 晚期大眾 | 主流市場 | 成熟與標準化，比早期大眾更保守 |
| 落伍者 | 主流市場 | 對新事物無感，能不用就不用 |

### 操作步驟

1. **確認目前位置** — 已有的採用者是誰：追求新事物的人（創新者、早期採用者），還是主流大眾？被早期愛好者喜歡不等於會被主流接受——PDA 的教訓。
2. **描出五個區隔的輪廓** — 對這個對象，每個區隔的人是誰、在意什麼、去哪找得到他們。寫得出來才攻得了。
3. **正視鴻溝** — 鴻溝卡在早期採用者與早期大眾之間，兩邊要的東西本質不同：前者追求新事物，後者想要完整方案和便利。誠實回答：目前的東西對早期大眾來說，完整嗎？方便嗎？有他們信得過的實績嗎？
4. **擬跨溝策略** — 挑一個灘頭堡區隔集中攻克，不要同時討好所有人：把產品補成該區隔要的「完整方案」，在區隔內累積口碑與實績，站穩後再逐步攻克下一個區隔。
5. **設時間窗與檢核訊號** — 不能在時間內跨越就進不了主流。定出跨溝的時間窗與檢核訊號（早期大眾的實際採用證據），到期未過就回頭修產品或換灘頭堡。

### 常見錯誤

- 把早期熱度當成功：愛好者的掌聲不代表主流買單，這正是跌入鴻溝的起手式。
- 用同一套訴求打全部區隔：早期市場吃「新」，主流市場吃「完整與便利」，話術與產品都要換。
- 假設採用曲線是平滑的：以為自然會從早期滑進主流，忽略鴻溝的存在。
- 同時討好所有區隔：資源分散，哪個都沒攻克。逐步來。
- 產品不完整就衝主流：早期大眾要完整方案，半成品只會燒掉口碑。
- 沒有時間感：鴻溝有時間窗，拖太久熱度散、資源盡，就跌進去了。

### 自檢清單

做完鴻溝理論分析後逐項檢查，任何一項不成立就回去補：

- [ ] 判斷過目前採用者屬於哪個區隔，沒把早期熱度當主流證據
- [ ] 五個區隔對這個對象各自是誰、在意什麼，寫得出來
- [ ] 對早期大眾檢查過：完整嗎、方便嗎、有可信實績嗎
- [ ] 有選定灘頭堡區隔，不是同時討好所有人
- [ ] 有把產品補成完整方案的具體計畫
- [ ] 有跨溝時間窗與檢核訊號，到期未過的應變也想好
- [ ] 攻克順序是逐步的，一個區隔站穩再攻下一個

## 長尾模型：利基市場還是有利可圖

「八二法則」說 80% 的營收來自 20% 的暢銷商品。但《連線》（Wired）雜誌總編輯克里斯・安德森（Chris Anderson）用需求曲線說明：暢銷品只是市場的主體 20%，把其餘 80% 品項的獲利加總起來，其實比暢銷品來得高，稱為「長尾模型」。意思是發展可以更多元——過去認為無利可圖的利基市場，集結起來的價值能超過大眾市場。一句話：**利基市場還是有利可圖。**

任何「品項多、分布不均」的東西都適用：產品線、內容庫、功能清單、客群、關鍵字。

| 區塊 | 佔品項 | 特性 |
| --- | --- | --- |
| 主體（大眾市場） | 約 20% | 單品銷量高，傳統上的暢銷主力 |
| 長尾（利基市場） | 約 80% | 單品銷量低，但加總價值可能超過主體 |

### 操作步驟

1. **畫需求曲線** — 把品項按銷量或使用量排序，看頭尾分布：主體 20% 貢獻多少、長尾 80% 各自多小。
2. **算尾巴的集結價值** — 別只看單品。長尾的單品都小，但把尾巴全部加總，跟頭部比一比——過去認為無利可圖的利基，集結起來可能超過大眾市場。
3. **檢查長尾成立條件** — 尾巴要划算有兩個前提：上架與維持尾巴品項的邊際成本夠低（無限貨架、自動化通路），而且需求找得到尾巴（搜尋、推薦、分類導航）。成本高又沒人找得到，尾巴只是庫存，八二法則主導，聚焦頭部。
4. **決定頭尾配置** — 條件成立就經營多元尾巴、集結利基；不成立就聚焦頭部暢銷品。多數情境是混合：頭部帶流量與聲量，尾巴賺加總。
5. **持續觀察頭尾流動** — 尾巴品項可能竄升成頭部，頭部會老化滑進尾巴（接 S 曲線的階段判斷）。定期重畫分布、調整配置。

### 常見錯誤

- 只看單品營收就砍尾巴：長尾的價值在加總，單品視角永遠覺得尾巴不值。
- 無條件信長尾：貨架有限、品項有固定維持成本的情境，尾巴是負擔。先檢查成立條件。
- 頭尾二選一：多數情境是頭部帶流量、尾巴賺集結，不是互斥。
- 尾巴沒人找得到：沒有搜尋、推薦、導流機制，長尾只是長庫存。
- 把八二法則當指令：「80% 營收來自 20% 商品」是現狀描述，不是「砍掉八成品項」的指示，先算過尾巴的集結價值再決定。
- 把長尾當成什麼都做：多元不等於無限展開，尾巴品項仍要邊際成本夠低才留。

### 自檢清單

做完長尾模型分析後逐項檢查，任何一項不成立就回去補：

- [ ] 品項有按銷量或使用量排序，畫出頭尾分布
- [ ] 長尾的集結價值算過，和頭部比較過
- [ ] 成立條件檢查過：尾巴邊際成本夠低、需求找得到尾巴
- [ ] 頭尾配置有明確決定，說得出依據
- [ ] 沒有只憑單品視角砍尾巴，也沒有無條件擴尾巴
- [ ] 有讓需求找到尾巴的機制（搜尋、推薦、分類）
- [ ] 有定期重看頭尾流動的安排
