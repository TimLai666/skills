# 邏輯推導

## 主題定位

邏輯的定義是「依照事物的道理思考」，而不是一直困在「我該不該這麼做」「到底要怎麼選」的原地打轉（《邏輯思考法》）。《杜拉克談高效能的5個習慣》指出：懂得邏輯思考，能判斷問題的根本原因，才能有效率地檢視解決方案是否真正對應到問題。

套用到任何任務上：先把問題與成因結構化，結論才有道理可循，後續的方案、內容或主張才對得上真正的問題。

## MECE：相互獨立、完全窮盡

麥肯錫顧問公司延伸出的原則（mutually exclusive, collectively exhaustive）。相互獨立：所有元素彼此不重疊；完全窮盡：重要項目一個不漏、資訊不遺漏。一句話：**列舉需獨立且窮盡，確保思考周全。**

任何需要「把一個整體拆成幾個部分」或「列舉一組東西」的時刻都用它把關：拆解問題成因、列出方案選項、切分章節或段落結構、劃分模組職責、盤點對象或情境。

### 操作步驟

1. **界定整體** — 白紙黑字寫下要窮盡的整體是什麼：「我要討論哪些問題」、這個集合的邊界在哪。整體沒有定義，就無從判斷遺漏。
2. **選一個切入維度** — 決定這一層用哪一個標準來切，例如用時效性、重要程度作為衡量指標。常用的切法：
   - 流程／時間：事前、事中、事後，或按階段切
   - 組成／結構：整體等於部分之和（元件、段落、模組）
   - 公式／因子：把結果拆成可相加或相乘的因子
   - 對立二分：內／外、可控／不可控、主觀／客觀、質／量
   - 對象分群：按涉及的人、使用情境、來源切

   同一層只能用一個維度，混用維度是「重複」最常見的來源。
3. **展開分類項** — 依選定的維度列出各部分，每一項的抽象層級要一致、涵蓋寬度要相當。
4. **檢查相互獨立（ME）** — 兩兩互看：有沒有一項其實是另一項的子集、成因、或只是換句話說？能不能舉出一個具體案例同時落進兩格？可以的話就是有重複，回到步驟 2–3 重切或合併。
5. **檢查完全窮盡（CE）** — 問「除了這些，還有嗎？」拿幾個真實或想像的具體實例逐一歸位，歸不進任何一格的就是遺漏；再檢查有沒有哪一項「太窄」，只代表它該涵蓋類別的一小塊。
6. **修正並定案** — 合併重複、拉齊抽象層級、放寬太窄的項目、補上遺漏。窮盡的標準是「重要項目不遺漏」，不是無限列舉；拆到足以支撐當前任務的判斷就停。

### 對照範例：顧客抱怨店員態度不佳

| 拆法 | 內容 | 判定 |
| --- | --- | --- |
| MECE | 太晚回應／溝通不良／服務細節不好／價值觀落差 | 沒有遺漏、沒有重複 |
| 非 MECE | 太晚回應／在忙別的客人／臉色不好／價值觀落差 | 「在忙別的客人」是「太晚回應」的成因，層級混在一起 → 重複；「臉色不好」只是服務細節的一小塊 → 太窄；其他服務細節沒有任何一格能接 → 遺漏 |

### 常見錯誤

- 沒界定整體就開始列，列完才發現對範圍的認知各說各話。
- 同一層混用兩個維度，項目自然互相重疊。
- 抽象層級不一致，把「原因的原因」和「原因」擺在同一層。
- 用一個大而無當的「其他」湊窮盡。「其他」如果裝了很多重要的東西，代表維度選錯了，重切。
- 追求絕對窮盡、無限展開。窮盡以當前任務的重要項目為準。
- 分類完就停。MECE 是把思考變周全的手段，分完要回頭回答原本的問題。

### 自檢清單

做完 MECE 後逐項檢查，任何一項不成立就回去補：

- [ ] 要窮盡的整體有白紙黑字寫下來，邊界清楚
- [ ] 這一層只用一個維度，說得出維度是什麼
- [ ] 任兩項之間舉不出同時落進兩格的具體案例
- [ ] 沒有任何一項是另一項的子集、成因或換句話說
- [ ] 各項抽象層級一致、涵蓋寬度相當
- [ ] 拿至少三個具體實例測試，全部歸得進某一格
- [ ] 問過「除了這些還有嗎」，重要項目沒有遺漏
- [ ] 沒有用大而無當的「其他」湊窮盡
- [ ] 分類結果有回頭回答原本的問題

## 空雨傘：事實、解釋、行動

麥肯錫顧問公司提出的「天空、雨、傘」思考法，引導出正確行動。天空是問題的事實（現在烏雲密布），雨是對問題的解釋（很多烏雲，所以可能會下雨），傘是解決問題的實際行動（會下雨，所以帶傘）。遇到 A 現象，就去聯想關於 A 的解釋，再找出因應的行動方案。一句話：**分析現狀預示的未來，及早採取行動。**

任何「看到現象要決定怎麼辦」的時刻都適用：解讀資料或回饋、判讀異常狀況、從觀察導出建議、把分析收斂成下一步。與預期有落差的現象（以為會塞車卻沒塞）是最值得跑空雨傘的訊號。

### 操作步驟

1. **空：確認事實** — 寫下觀察到的現況。只放直接觀察到或查證過的事實，不夾帶推測與評價；句子裡若出現「應該、大概、我覺得」，就搬到下一層去。
2. **雨：解釋事實** — 問「這個事實為什麼發生？它預示接下來會怎樣？」列出想得到的合理解釋，每個解釋都必須能從第 1 步的事實推出來。有多個解釋時，挑依據最強的當主解釋，並標明這是推論不是事實。
3. **傘：決定行動** — 根據主解釋回答「所以現在該做什麼？」行動要具體可執行，並且趕在預示的事情發生之前。同一個解釋對不同立場的對象會導出不同行動（返鄉潮延後：小家庭該立即返鄉，休息站該加強進貨），所以行動要寫明誰、做什麼、何時。
4. **檢查三層連貫** — 逐層驗證：解釋拿掉事實就不成立（雨依賴空）；行動拿掉解釋就沒理由做（傘依賴雨）。若某個行動跟任何解釋都對不上，就是從事實直接跳到行動，回頭補推理。

### 對照範例：年假國道車流

| 層 | 內容 |
| --- | --- |
| 空（事件） | 年假期間，國道沒有想像中塞車。 |
| 雨（解釋） | 天氣不好，大家都晚 1～2 天返鄉——車潮沒有消失，只是延後。 |
| 傘（行動） | 小家庭→要立即返鄉，要不然愈晚會愈塞。國道休息站→加強進貨，以應付人潮最多的時候。 |

### 常見錯誤

- 「空」混入意見：把推測、評價寫成事實，後面整串推理跟著歪。
- 從空直接跳到傘：略過解釋層，行動變成反射動作，常對不上真正的問題。
- 雨與空脫鉤：解釋不是從事實推出來的，而是先有成見再找事實背書。
- 只想到一個解釋就行動：沒有比較過其他解釋哪個依據更強。
- 有空有雨沒傘：分析完不給行動。空雨傘的目的就是引導出行動，停在解釋等於白想。
- 行動不分對象：沒寫清楚誰該做什麼，人人有責等於沒人負責。

### 自檢清單

做完空雨傘後逐項檢查，任何一項不成立就回去補：

- [ ] 「空」的每一條都是可查證的事實，沒有夾帶推測或形容
- [ ] 「雨」的每個解釋都能從「空」推出來，且標明是推論
- [ ] 至少想過一個替代解釋，說得出為什麼採用目前的主解釋
- [ ] 解釋有推到「接下來會怎樣」，不是只重述現在
- [ ] 「傘」的行動直接回應解釋，拿掉解釋行動就失去理由
- [ ] 行動寫明誰、做什麼、何時做，具體可執行
- [ ] 行動趕在預示的事情發生之前，不是事後補救
- [ ] 沒有從事實直接跳到行動的跳層
