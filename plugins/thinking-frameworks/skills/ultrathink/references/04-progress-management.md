# 進度管理

## 主題定位

把焦點放在設定目標、優化流程、控制進度，就能盡量避免認知落差、進度拖延等人為失誤，交出高準確性、高完成度的成果。《思維的良率》也提到，流程管理是企業成長的指標之一：運用思考工具檢視內部流程，常常能抓出可以再改進的地方，無形之中也就改善了工作品質。

套用到任何任務上：交付物的品質不只來自內容本身，也來自目標設得對、流程走得穩、進度控得住。動手前先把目標立正確，執行中把流程與進度管好。

## 惠特默模型：檢核目標有效性的 14 項原則

藝術家米開朗基羅曾說：「大多數人最大的危機不在於目標設得太高而達不到，而是太低而輕易達成。」沒有挑戰性的目標，就算達成了，也不能令人進步。職場教練約翰・惠特默（John Whitmore）在《高績效教練》整理 14 項指標，簡稱 SMART、PURE、CLEAR 原則：寫下目標後，逐一檢討模型的每一個要求，符合原則再執行，才有效益。一句話：**檢核目標有效性的 14 項原則。**

任何要設定目標的時刻都適用：專案、任務、學習、寫作計畫；別人交付的目標也能拿來驗收。三組原則各管一面：SMART 管目標的形狀、PURE 管目標的表述、CLEAR 管目標的張力與基礎。

| 組 | 14 項指標 |
| --- | --- |
| SMART | 具體 specific、可衡量 measurable、可達成 attainable、務實的 realistic、時間階段 time-phased |
| PURE | 正面的陳述 positively stated、了解的 understood、相關的 relevant、道德的 ethical |
| CLEAR | 具挑戰性 challenging、合法 legal、環保的 environmentally、認同的 agreed、有紀錄的 recorded |

### 操作步驟

1. **寫下目標** — 白紙黑字一句話：做什麼、做到什麼程度、什麼時候。沒寫下來就無從逐項檢討，CLEAR 的「有紀錄」從這裡開始。
2. **過 SMART：目標的形狀** — 具體嗎？可衡量嗎（怎麼知道達成了）？可達成嗎？務實嗎（資源與現實條件撐得起）？有時間階段嗎（期限或分段節點）？
3. **過 PURE：目標的表述** — 是正面陳述嗎（寫要達成什麼，而不是「不要怎樣」）？相關的人都了解嗎？跟大局相關嗎？合乎道德嗎？
4. **過 CLEAR：目標的張力與基礎** — 具挑戰性嗎（太低而輕易達成，達成了也不會進步）？合法嗎？對環境與周遭沒有負面影響嗎？相關的人認同嗎？有紀錄嗎？
5. **不符就改寫，改完重檢** — 任何一項不過，回頭改寫目標再跑一遍。14 項全過才執行，符合原則再執行，才有效益。

「可達成、務實」與「具挑戰性」不衝突：目標要落在「跳一下搆得到」的區間。太低沒進步，太高變空話。

### 常見錯誤

- 目標設太低：輕易達成、沒有進步。米開朗基羅警告的正是這個——多數人的危機不是設太高。
- 把「不要做什麼」當目標：負面陳述。改成正面陳述，寫要達成什麼。
- 不可衡量：「做好一點」「盡快完成」，沒有判準，達沒達成全憑感覺。
- 沒有時間階段：沒期限的目標會無限延後。
- 把挑戰性與可達成當二選一：其實要同時滿足，落在跳一下搆得到的區間。
- 只有自己懂、只有自己同意：了解與認同是兩道獨立關卡，相關的人缺一道，執行時就散掉。
- 檢核完不改寫：發現不符卻照原樣執行，檢核白做。

### 自檢清單

寫下目標後逐項檢核，任何一項不成立就改寫目標、重新檢核：

- [ ] 目標已白紙黑字寫下
- [ ] 具體（specific）：講得出做什麼、到什麼程度
- [ ] 可衡量（measurable）：有客觀判準知道達成與否
- [ ] 可達成（attainable）：做得到
- [ ] 務實（realistic）：符合現有資源與現實條件
- [ ] 時間階段（time-phased）：有期限或分段節點
- [ ] 正面的陳述（positively stated）：寫要達成什麼，不是「不要怎樣」
- [ ] 了解的（understood）：相關的人都了解
- [ ] 相關的（relevant）：與大局相關，不是為設而設
- [ ] 道德的（ethical）：目的與手段都合乎道德
- [ ] 具挑戰性（challenging）：不是輕易達成的低標
- [ ] 合法（legal）
- [ ] 環保的（environmentally）：不對環境與周遭造成負面影響
- [ ] 認同的（agreed）：相關的人都認同
- [ ] 有紀錄的（recorded）：目標與檢核結果都留有紀錄
- [ ] 不符的項目都改寫過並重新檢核

## KPI 樹狀圖：分解大目標，成為可逐步達成的子目標

個人或組織訂立好的最終目標，就是關鍵目標指標（KGI，key goal indicator）。為順利達陣，還需要制定階段性目標，亦即關鍵績效指標（KPI，key performance indicator），監控目標達成度、在問題的第一時間盡快改善。繪製時把 KGI 拆解為數個小 KPI，以「加減乘除」呈現，釐清目標間的關聯性，進一步驗證。一句話：**分解大目標，成為可逐步達成的子目標。**

任何「大目標不知從何下手」的時刻都適用：營運指標、專案進度、學習量、產出量，只要最終目標可衡量就拆得動（先過惠特默模型的目標自然合格）。它跟心智圖不同：心智圖是聯想的發散，KPI 樹是算式的拆解——每個子節點跟母節點之間必須是加減乘除的數學關係。

### 操作步驟

1. **定 KGI** — 寫下最終目標的衡量指標：指標名稱、單位、目標值。KGI 本身必須可衡量，拆解才有意義。
2. **用加減乘除拆第一層** — 問「KGI 由什麼組成」：能相乘的拆乘法（營業額＝客戶數 × 客單價），能相加的拆加法（來店人數＝新客戶＋老客戶）。每個子指標都標上單位。
3. **逐層往下拆** — 對還不能直接行動的 KPI 繼續拆，直到葉節點是「可以直接監控、有人能對它採取行動」的指標。
4. **驗證關聯性** — 兩道檢查：數學上，每個節點的算式成立嗎？單位相乘除對得起來嗎（人 × 新台幣/人 ＝ 新台幣）。結構上，加法拆解的子項用 MECE 檢查，不重不漏。
5. **用樹監控與改善** — 為各 KPI 設定目標值與監控頻率。KGI 落後時，沿樹往下找是哪個 KPI 掉了，在問題的第一時間盡快改善：行動打在葉節點，效果沿樹傳回 KGI。

### 對照範例：營業額

```
營業額（新台幣）＝ 客戶數（人）× 顧客客單價（新台幣/人）
├─ 客戶數（人）＝ 購買率（%）× 來店人數（人）
│   └─ 來店人數（人）＝ 新客戶（人）＋ 老客戶（人）
└─ 顧客客單價（新台幣/人）＝ 產品單價（新台幣/個）× 平均每人購買數（個/人）
```

每一層的單位都對得起來（% × 人＝人；新台幣/個 × 個/人＝新台幣/人），這就是「進一步驗證」的具體做法。營業額掉了，往下看是購買率、來店人數還是單價在掉，行動立刻有落點。

### 常見錯誤

- KGI 不可衡量就開拆：目標沒有單位與數值，整棵樹是空中樓閣。
- 拆出來的是聯想不是算式：子項跟母項只是「有關」，沒有加減乘除關係。那是心智圖，不是 KPI 樹。
- 單位對不起來：相乘除後的單位不等於母項單位，算式必錯。
- 加法拆解重疊或遺漏：子項分不乾淨，數字會重算或漏算。
- 拆到不可行動的葉子就停：沒有人能對葉節點做事，監控了也不知道怎麼改。
- 只畫不監控：沒設目標值、沒定頻率，問題無法在第一時間發現。
- 衝單一 KPI 傷整體：KPI 是手段，KGI 才是目的；某節點的行動若犧牲其他節點，對照全樹再取捨。

### 自檢清單

做完 KPI 樹狀圖後逐項檢查，任何一項不成立就回去補：

- [ ] KGI 明確：指標、單位、目標值都有
- [ ] 每一層拆解都是加減乘除的算式關係，不是模糊聯想
- [ ] 每個節點都標了單位，相乘除後單位對得起來
- [ ] 加法拆解的子項用 MECE 檢查過，不重不漏
- [ ] 葉節點都是可直接監控、可採取行動的指標
- [ ] 各 KPI 有目標值與監控方式，KGI 落後時能沿樹定位問題
- [ ] 行動打在葉節點上，且沒有犧牲其他節點或 KGI
