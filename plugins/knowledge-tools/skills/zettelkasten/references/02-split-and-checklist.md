# 拆卡判準與變更檢核清單

> llm-wiki 的 SKILL.md（Note Discipline 節）內嵌了本檔清單與判準的降級核心版；
> 修改本檔的判準或清單時，須同步該處。

## 拆卡判準

命中任何一條就該拆：

1. **標題測試** — 一句陳述句寫不完這張卡在講什麼，或必須用「與、以及、和」串接。
2. **獨立引用測試** — 卡中某段落可以被另一個脈絡單獨引用，而不需要帶上整張卡。
3. **多問題測試** — 這張卡回答了超過一個問題。
4. **變動速率測試** — 卡的某一半會隨新來源更新，另一半穩定不動。更新節奏不同的
   內容屬於不同想法。
5. **連結歧義測試** — 別的卡連過來時，你說不清它到底是連到卡中哪個論點。

尺寸不是判準，只是警訊：卡片很長（如超過 200 行）值得跑一次上述測試，
但長卡若真的只有一個想法就不拆，短卡若混了兩個想法照樣拆。

## 不拆的判準（避免拆過頭）

以下情況不要拆：

- 拆出來的卡單獨讀不成立（失去自主性）。主張與它成立的關鍵前提是一體的。
- 拆出來的卡之間必然永遠一起被引用，從無單獨引用的場景。
- 只是同一個想法的不同措辭或不同例子。例子併入原卡或作為卡內小節。

原子性的單位是「想法」不是「句子」。拆到一句話一張卡是走火入魔。

**邊界案例**：`提取練習比重讀更能鞏固記憶.md` 裡有三個實驗（Karpicke 2006、
Roediger 2008、課堂應用觀察），卡片頗長。跑判準：三個實驗都在支撐同一個主張，
沒有任何一個會被其他脈絡單獨引用，標題一句話寫得完 → 不拆，在
`split_decisions` 記錄「長但單一想法，實驗屬證據小節」。

## 實體卡的拆卡

五條判準同樣適用，但原子性的單位換成「實體」：

- 實體的某個面向長成獨立論點（例如從颶風基科的頁面長出「東太平洋颶風的登陸
  預報誤差偏大」的分析）→ 論點拆成概念卡，實體卡在提到該面向的句子上就地連結。
- 多個實體重複出現同一個現象（好幾場颶風都經歷快速增強）→ 這是概念卡的訊號：
  把現象寫成概念卡，各實體卡在內文提到該現象處就地連過去。
- 兩個實體糾纏在同一張卡（颶風基科與同季另一場颶風寫在一起）→ 一卡一實體，
  拆開互連。

## 拆卡操作步驟

1. 列出這張卡實際涵蓋的想法，每個想法一句陳述句。
2. 每個想法建一張新卡：陳述句標題（拆出實體則用實體名稱）、自己的話、自主成立。
3. 新卡之間互連：內文互相提到處就地連結；內文沒自然提到的關係補在文末並說明。
4. 處理原卡，二選一：
   - 內容全數搬走 → 原卡改寫為摘要＋連結（實質變成小型結構筆記），或直接刪除
     並把入連結改指到對應新卡。
   - 保留主想法 → 拆出去的部分在原卡留一行摘要＋連結。
5. 修復入連結：找出所有連到原卡的卡片，逐一判斷該改連到哪張新卡；改指時
   檢查顯示別名是否仍與新目標的主張相符，不符就改寫。
6. 更新相關結構筆記與索引。

## 變更檢核清單（勾選版）

每次卡片 create / update，寫入前逐項勾：

```markdown
- [ ] 原子性：這次寫入的是單一想法（實體卡：單一實體）；寫入後這張卡仍只涵蓋一個
- [ ] 拆卡：跑過五條拆卡判準，命中的已拆；邊界案例已記錄不拆理由
- [ ] 重複與矛盾：搜尋過既有卡片；已存在→更新既有卡不開新卡；已存在且互相
      衝突→不併寫覆蓋，寫成新卡與既有卡互連標明分歧，並依該庫慣例標記矛盾狀態
- [ ] 連結：內文提到其他卡片主題的詞已就地連結（至少 1-2 條出連結）；
      內文沒提到的重要關係已補在文末；被連卡已判斷是否回連
- [ ] 結構：相關結構筆記／索引已判斷是否需要收錄或更新
- [ ] 自主性：這張卡單獨讀成立，不依賴來源脈絡
- [ ] 自己的話：內容是改寫不是摘抄；出處已標
```

任何一項勾不了就先處理再寫入，不要「先寫了之後再補」。
因回連而觸發的被連卡編輯只處理連結項，不重跑全清單、不再遞迴觸發下一層回連判斷。

## 範例：一次拆卡

原卡 `間隔重複.md`（混了三個想法）：

> 間隔重複比集中複習有效，因為提取練習本身會強化記憶。Anki 是最常用的間隔重複
> 軟體，用 SM-2 演算法排程。不過間隔重複只適合原子化的知識點，對需要整體理解的
> 概念效果差。

標題測試失敗（「間隔重複的效果、工具與限制」需要頓號），拆成三張：

1. `間隔重複比集中複習有效.md` — 機制。內文講到適用邊界的那句話就地連結
   [[間隔重複只適合原子化知識點]]。
2. `間隔重複只適合原子化知識點.md` — 適用邊界與原因。內文交代背景時就地連結
   [[間隔重複比集中複習有效]]。
3. `anki.md` — 實體卡（實體名稱標題）。內文說明它以 SM-2 演算法排程、
   實作的機制處就地連結 [[間隔重複比集中複習有效]]。

原卡刪除，入連結依語境分別改指到 1 或 3。

注意卡 2 拆出來仍自主成立（卡內一句話交代了「間隔重複靠提取練習強化記憶」的
背景並連結卡 1），這就是「拆而不失自主性」。

## 範例：更新既有卡時的檢核

拆卡判斷不只發生在新增，更新既有卡時最容易漏。兩個對照案例：

### Case A：更新就是更新

讀到 Karpicke 2006 的實驗，想補進 `提取練習比重讀更能鞏固記憶.md`。
跑檢核：實驗是同一主張的新證據，不是新想法（判準全未命中）→ 更新原卡：
補證據小節、`source` 加上新文獻筆記、bump `updated`。

### Case B：更新觸發拆卡

讀到「提取難度要適中，太難的提取會失敗且帶來挫折」，直覺想補進同一張卡。
跑檢核：這回答的是「提取的難度該怎麼設計」，不是「提取是否比重讀有效」→
命中多問題測試；而且「適度困難」會被學習設計、題目出題等其他脈絡單獨引用 →
命中獨立引用測試。處理：新開 `提取難度適中時學習效果最好.md`，新卡內文交代
前提時就地連結 [[提取練習比重讀更能鞏固記憶]]；原卡在提到難度的那句話上
連向新卡。

兩個案例的差別不在內容長短，在「是否回答了新的問題」。
