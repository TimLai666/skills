# 快速啟動指南

> 如果你只有 30 分鐘，照這個跑。

## 30 分鐘速成版（單篇貼文）

### Step 1（5 分鐘）：定位確認

回答 4 題：
- 你是誰 / 你的品牌是什麼？
- 你的 TA 是誰？（具體年齡 + 身份 + 情境）
- 你想達成什麼？（漲粉 / 留言 / 品牌印象 / 課程作業）
- 不能講什麼？

### Step 2（5 分鐘）：快查趨勢

```
□ 打開 Threads → 看當前前 5 個趨勢話題
□ web_search 兩個關鍵字組合（你的 TA + 近期月份）
□ 看 1-2 個同類爆文，記 Hook 句型
□ 寫一句「今天 TA 最有感的是什麼」
```

### Step 3（5 分鐘）：選題材 + 情緒

從趨勢中選 **1 個** 題材（不是 3 個），快速判定：
- TA 看到會點頭嗎？
- 跟品牌連得起來嗎？
- 至少能預想 3 種留言嗎？

選 **1 個主導情緒**：共鳴 / 憤怒 / 幽默 / 驚訝 / 焦慮 / 溫暖。

### Step 4（5 分鐘）：寫 Hook + 結構

從 `references/03-hook-library.md` 找對應情緒的 Hook 類型，寫 3 個 Hook，選最好的。

從 `references/04-post-formulas.md` 選一個結構：
- 不知道選什麼 → 先用「共鳴型」
- 想衝留言量 → 用「爭議型」

### Step 5（5 分鐘）：寫貼文

```
[Hook（≤ 25 字）]
[空行]
[2-3 段鋪陳，短句]
[空行]
[一句核心觀點]
[空行]
[留言鉤：問句 / 立場選擇]
```

### Step 6（3 分鐘）：自檢 + 圖卡 prompt

跑自檢清單：
```
□ Hook ≤ 25 字
□ 字數 ≤ 350
□ 沒有外部連結在內文
□ 一個 hashtag 或不用
□ 結尾有留言鉤
□ 主導情緒清楚（一個）
□ 能預想 5 種留言
```

如果要圖卡，產出 prompt（見 `references/08-card-design.md`）。

### Step 7（2 分鐘）：時間 + 互動劇本

- 建議時段：[週X 20:00]
- 第一則自留言要寫：[內容]
- 預想前 5 則留言：[列出]

## 完整輸出範例（給使用者參考）

```
─────────────────────────────────────
✅ 貼文已準備好
─────────────────────────────────────

【貼文正文】（直接複製）
─────────────────────────────────────
很多人以為行銷系只是在做海報。

明明大三作業最後一頁是「附錄」，
不是「結語」。
我們學的不是「會做東西」，
是「為什麼要做這個東西」。

行銷的真本事，
是研究人為什麼停下來。

你以為大學行銷系都在學什麼？
─────────────────────────────────────

【建議發文時段】
週四 20:00（本週最佳）

【第一則自留言】
其實這篇是有點生氣寫的。
被問了第 N 次「行銷系是不是會做美編」之後。
你被問過什麼讓你想翻白眼的問題？

【預想留言】
1. 「我們系也常被誤會 +1」
2. 「我以為行銷就是打廣告？」
3. 「拜託，廣告系才會做廣告好嗎」
4. 「真的，我家人到現在還覺得我會用 PS」
5. 「那行銷系到底在學什麼？認真問」

【發文後劇本】
T+1   發第一則自留言
T+5   找 2 個朋友去留有觀點的留言
T+10  回覆所有早期留言
T+15  到某個大學 Threads 帳號近期貼文下留「觸」的留言

【圖卡 Prompt】
主標：很多人以為 / 行銷系 / 只是在做海報
副標：真正的行銷，是研究人為什麼停下來
版型：4:5 黑底白字，Apple Keynote 風格
─────────────────────────────────────
```

## 常見急用情境

### 情境 A：「我明天要交課程作業，要 2 則貼文」

→ Step 1-7 跑兩次，每篇 30 分鐘。
→ 用不同 Hook 類型（一個共鳴 + 一個爭議，互補）。
→ 圖卡用同一風格（保持品牌一致）。

### 情境 B：「我要為品牌建立帳號，從 0 開始」

→ 先讀 `references/09-brand-voice.md` 定品牌語氣（30 分鐘）
→ 用 `assets/content-calendar-template.md` 規劃第一個月（30 分鐘）
→ 每篇用本指南產出（30 分鐘 × 12 篇 = 6 小時）
→ 一週完成第一個月的內容庫

### 情境 C：「我有現成題材，只想要 polish」

→ 跳過 Step 1-3
→ 直接從 Step 4（Hook 優化）開始
→ 重點檢查：Hook 強不強、結尾留言鉤有沒有

### 情境 D：「我發了一篇沒人理，想知道為什麼」

→ 對著 `references/10-pitfalls.md` 逐項檢查
→ 通常是 Hook、外連、留言鉤、時段四項之一出問題
→ 不要刪原文，下一篇調整

## 給長期用戶的進階建議

1. **每篇都填 `assets/post-template.md`**：累積資料才能優化
2. **每週看 `assets/analytics-template.md`**：找規律
3. **每月做月度總結**：調整方向
4. **建立靈感倉**：日常累積素材
5. **持續學趨勢**：每天 10 分鐘看 Threads 趨勢話題
