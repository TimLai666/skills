## 90 天累積觀察（找出真正的「我的爆文公式」）

累積 90 天（約 60-90 篇貼文）後，做一次大檢討：

### 我的爆文 DNA（前 10 大表現最好的貼文）

```
排名 | Hook 類型 | 結構 | 情緒 | 主題 | 時段 | 觸及
1.   |          |     |      |     |     |
2.   |          |     |      |     |     |
3.   |          |     |      |     |     |
4.   |          |     |      |     |     |
5.   |          |     |      |     |     |
...
```

找出**共同特徵**：

```
我的爆文公式 = 
  [最常出現的 Hook 類型] × 
  [最常出現的結構] × 
  [最常出現的情緒] × 
  [最常出現的時段]
```

### 我的爛文 DNA（前 10 大表現最差的）

```
排名 | Hook 類型 | 結構 | 情緒 | 主題 | 時段 | 觸及
1.   |          |     |      |     |     |
2.   |          |     |      |     |     |
...
```

找出**地雷區**：哪些組合是你個人 / 品牌不該走的。

## Threads 內建數據查看

Threads App 內可看的數據：
- 個人頁右上角 → 互動數據（Insights）
- 每篇貼文點「···」→ 查看觸及與互動細節

第三方工具：
- Buffer：Threads 數據整合
- Hootsuite：跨平台分析
- Metricool：含詳細圖表

---

## 給使用者的建議

1. **不要頻繁看數據**：發文後 24 小時內不停刷會焦慮
2. **48 小時是觀察黃金窗口**：之後增長放緩，可以記錄
3. **不要只看單篇**：要看「移動平均」找規律
4. **質性比量化重要**：「留言區在聊什麼」比「觸及多少」更能優化內容
5. **失敗也是資料**：爛文也要紀錄，知道「不該做什麼」

---

## 數據迷思警告

❌ 「觸及高就是爆文」
✅ 觸及只是觸發，**留言/觸及比** 才是品質指標

❌ 「讚多就是好內容」
✅ 讚是最低權重的互動，留言和引用才是

❌ 「發爆文就會漲粉」
✅ 不一定。Threads 演算法常推「一次性互惠」貼文，要看**個人頁點擊**和**新增追蹤**

❌ 「同樣 Hook 一定再爆一次」
✅ Threads 是即時平台，3 天前的梗今天可能就死了
