# 10｜避雷清單｜常見死法

> 不知道為什麼會死的貼文，就是這些原因。

## 11 種「保證沉到底」的貼文死法

### 1. ❌ Hook 是介紹語

**死法範例**：
```
今天想跟大家分享行銷系的學習心得
讓我們一起來看看
```

**為什麼死**：
- 「今天想分享」「讓我們」「來看看」這些詞 = 自殺
- 讀者看到立刻知道是「準備來教學了」直接滑

**怎麼救**：
```
✅「行銷系第一年，我以為要學的是創意。
   結果學的是怎麼跟組員溝通。」
```

直接把觀點當 Hook。

---

### 2. ❌ 內容像 LinkedIn

**死法範例**：
```
身為一位行銷系學生，我認為品牌應當建立差異化的溝通策略，
透過深度洞察消費者行為，創造價值共鳴。
```

**為什麼死**：
- 沒人在 Threads 想看履歷
- 太多書面語、抽象名詞、長句

**怎麼救**：
- 把所有「應當」「身為」「透過」「進而」全刪掉
- 用人話：「我覺得」「我發現」「老實說」

---

### 3. ❌ 太多 Hashtag

**死法範例**：
```
[正文]
#行銷 #品牌 #大學生 #北市大 #社群 #創意 #職場 #學習 #成長
```

**為什麼死**：
- Threads 一篇限一個 hashtag
- 塞多個會被演算法降權，當垃圾文處理

**怎麼救**：
- 一個 hashtag 就好（最精準的那個）
- 或完全不用

---

### 4. ❌ 內文塞外部連結

**死法範例**：
```
完整內容看這 → https://medium.com/...
```

**為什麼死**：
- Threads 想留住使用者，內文有外連的貼文觸及降權
- 讀者也不會點，看到連結就滑走

**怎麼救**：
- 內文不提連結
- 第一則自留言放連結（Link in Comment）
- 或寫「留言區補充」「私訊我給你看」

---

### 5. ❌ 一篇講三件事

**死法範例**：
```
今天想聊三件事：
1. 行銷系是什麼
2. 我為什麼選這個系
3. 給高中生的建議

[然後每件事各 100 字]
```

**為什麼死**：
- 一篇貼文一個觀點，混合三個會稀釋
- 讀者抓不到「這篇在講什麼」就走

**怎麼救**：
- 拆成三篇單獨發
- 或者只講最重要的那一件

---

### 6. ❌ 純自吹自擂

**死法範例**：
```
我們的服務已經幫超過 1000 位客戶解決問題！
立刻聯絡我們了解更多 👉 [連結]
```

**為什麼死**：
- 純廣告文，Threads 演算法和讀者都討厭
- 完全沒有 hook、沒有觀點、沒有對話

**怎麼救**：
- 把成功案例變成**故事**
- 把「我們可以幫你」變成「你可能遇過...我也遇過」
- 用「客戶說的話」當 hook，而不是「我們的優勢」

---

### 7. ❌ 結尾沒有留言鉤

**死法範例**：
```
[一段很好的觀點]
[結束]
```

**為什麼死**：
- 讀者看完沒有「要回應」的衝動
- 留言量 = 0 = 演算法降權

**怎麼救**：
- 結尾留問題：「你覺得呢？」「你會選哪個？」
- 結尾留爭議：「我準備好被罵了」
- 結尾留共鳴：「你也是嗎？」

---

### 8. ❌ 標題黨但內容空

**死法範例**：
```
【震驚】這個秘密竟然沒人告訴你！
[然後內容很乾]
```

**為什麼死**：
- 讀者上當點開，發現沒料 → 立刻封鎖你
- 演算法會偵測「點進去後立刻滑走」當負面訊號

**怎麼救**：
- Hook 強，但**內容也要對得起 Hook**
- 不要為了流量犧牲品牌信任（一次失信幾個月補不回來）

---

### 9. ❌ 寫太長

**死法範例**：
```
[450 字滿滿的文字，密密麻麻沒有分段]
```

**為什麼死**：
- Threads 是手機平台，長密文字一秒被滑過
- 即使內容好，也讀不下去

**怎麼救**：
- 控制在 350 字以內（理想 150-250）
- 多分段、多空行
- 短句、口語化
- 真的需要長 → 拆成串文

---

### 10. ❌ 在錯時段發

**死法範例**：
```
週六早上 8 點發了爆款內容
週日晚上 11 點發了公告
```

**為什麼死**：
- 週末早上 / 週日晚上是 Threads 流量低谷
- 即使內容好，初期沒互動 = 演算法不推

**怎麼救**：
- 重要內容排到週二–週四 20:00–22:00
- 不能立即發 → 用排程

---

### 11. ❌ 發了就閃

**死法範例**：
```
T+0   發文
T+30  ❌ 還沒回任何留言
T+60  ❌ 還沒回
T+24 hr ❌ 終於上線回「謝謝大家」
```

**為什麼死**：
- 演算法看到「貼文有留言但發文者沒回」會判斷為低活躍
- 早期留言沒被回，後續留言量會掉

**怎麼救**：
- 發文後 15 分鐘內**保持在線**
- 每則早期留言都回（有內容地回，不是「謝謝」）
- 把回留言當成第二輪創作

## 5 種「踩線會炸」的內容雷區

### 雷區 1：政治對立

不要在沒有明確品牌定位「就是要講政治」的前提下，碰：
- 兩岸關係
- 統獨議題
- 政黨對立
- 選舉

**例外**：如果你的品牌就是政治評論帳號，那是另一個遊戲。

### 雷區 2：性別 / 種族 / 性向

開玩笑要極度小心。「男生都怎樣怎樣」「女生都怎樣怎樣」這種貼文：
- 短期可能爆，因為衝突大
- 長期會傷品牌，得罪一半受眾

### 雷區 3：嘲諷弱勢

- 嘲諷身障、失業、低收入、老人等
- 即使用「我覺得 OK 啊」當包裝，會被截圖到處傳

### 雷區 4：醫療 / 法律建議

- 不要在 Threads 給人「你應該怎麼治療」「你應該怎麼提告」
- 即使你是專業人士，也要強調「這不是專業建議」

### 雷區 5：未公開的商業資訊

- 不要洩漏前公司 / 客戶的內部資訊
- 不要把私下八卦變成公開貼文
- 截圖傳閱很快，覆水難收

## 業配 / 廣告貼文避雷

### ❌ 業配寫法

```
推薦這款新出的咖啡豆！
真的超好喝！
立刻購買 👉 [連結]
#業配 #咖啡 #推薦
```

**為什麼死**：
- 一秒辨識出是業配，互動率歸零
- 「真的超好喝」是業配文標準句

### ✅ 業配寫法

```
最近喝到一款豆子，
打破我「中焙才有層次」的偏見。

它是淺焙，
但喝起來像一首寫得很慢的詩。

我之前都不買淺焙，
因為怕「酸」這個字。
這款讓我重新理解，
酸不是缺點，是骨架。

[品牌名] 的 [產品名]，
最近在喝。

你有喝到讓你「打破偏見」的豆子嗎？
```

**為什麼活**：
- 用故事 / 觀點包裝產品
- 觀點是真的（即使有業配，觀點要真實）
- 結尾留討論空間

## 「補救」清單：發了之後發現不對怎麼辦

### 狀況 1：發完 15 分鐘沒任何互動

選項：
- **A**：自己留 2-3 則自留言（補充內容）試圖救
- **B**：請朋友幫忙留有觀點的留言
- **C**：直接放棄（不要刪，會傷演算法評分），下一篇調整

### 狀況 2：發完發現有錯字

Threads 發文 **15 分鐘內可以編輯**。超過就只能：
- 刪除重發（會失去初期觸及）
- 或在留言區補正：「上面打錯字，是『XX』不是『YY』」

### 狀況 3：發完發現觀點不對 / 引起反彈

- 不要立刻刪（會引起更大反彈）
- 補一則貼文 + 留言區回應
- 道歉要明確（不要「如果造成困擾」這種公關式語法）

### 狀況 4：意外爆紅了

- 把握流量：發第二則延伸貼文
- 在留言區跟讀者深度對話
- 千萬不要立刻發業配 / 推銷（會讓追蹤掉一半）
- 順勢介紹品牌：把個人頁簡介 / pinned post 準備好

---

## 給自己的最終提醒

> 「Threads 的演算法不會懲罰你犯錯，會懲罰你不真誠。」

每一篇貼文發出前，問自己一個問題：

**「如果只有 10 個人會看到這篇，我還會發嗎？」**

- 如果答案是「不會」 = 你在追逐流量，這篇通常會死
- 如果答案是「會」 = 你真的有東西要說，這篇有機會爆

爆文不是設計出來的，是**有什麼真的想說的東西**碰到了**對的結構與時機**。
