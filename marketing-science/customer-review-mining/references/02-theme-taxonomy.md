# Theme Taxonomy

## Rule

所有評論的第一層分類固定只用三大主題。這一層優先於理論標籤與 14 題項。

## 1. `service_experience`

定義：
- 交易、溝通、履約、支援、安裝協助、售後服務相關內容

常見子主題：
- 運送速度
- 包裝品質
- 溝通效率
- 顧客支持
- 問題解決

示例訊號：
- 「客服回很快」
- 「包裝破掉」
- 「賣家有幫我排除問題」

## 2. `product_performance`

定義：
- 產品本身的功能、品質、可靠性、適配性、外觀、安裝體驗

常見子主題：
- 品質良好
- 符合預期
- 尺寸適配
- 容易安裝
- 外觀設計

示例訊號：
- 「尺寸完全吻合」
- 「裝上去後效果有差」
- 「品質比預期差」

## 3. `value_perception`

定義：
- 顧客對價值、信任、價格合理性、再購意願與整體值得與否的評價

常見子主題：
- 高 CP 值
- 與描述一致
- 信任賣家
- 再購意願

示例訊號：
- 「這個價位很值得」
- 「下次還會買」
- 「跟頁面描述不一致」

## Classification Rules

- 每則評論至少分到一個主題
- 可多標籤，但不要超過最有證據的 2-3 個主題
- 若評論同時談客服與品質，主題可並列
- 無法分類時先標示不確定，不要硬塞
