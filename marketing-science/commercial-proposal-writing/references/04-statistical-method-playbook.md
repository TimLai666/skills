# Statistical Method Playbook

本手冊定義每個方法的固定輸出契約：

1. 適用情境
2. 輸入欄位
3. 假設前提
4. Python 範例
5. 商業解讀
6. 企劃落地句型

---

## 1) PCA / CFA（構面縮減）

### 適用情境

- 變數很多、需要萃取核心構面來做市場區隔或需求洞察。

### 輸入欄位

- 多個連續變數（行為、偏好、滿意度量表）。

### 假設前提

- 變數具共變關係。
- 樣本數至少為變數數量的 5-10 倍。

### Python 範例

- 參考 `scripts/pca_cfa_template.py`

### 商業解讀

- 說明哪些構面最能解釋客群差異（例如「價格敏感度」「速度需求」）。

### 企劃落地句型

- 「根據 PCA，客戶偏好可濃縮為 3 個核心構面，其中『即時性需求』解釋 41% 變異，故產品路線優先強化即時能力。」

---

## 2) KMeans Cluster（市場分群）

### 適用情境

- 已有客戶行為或屬性資料，需形成可操作的客群分群。

### 輸入欄位

- 可量化特徵欄位（頻次、金額、產品使用深度等）。

### 假設前提

- 特徵尺度可比較（需標準化）。
- 分群數 `k` 可用 elbow/silhouette 佐證。

### Python 範例

- 參考 `scripts/kmeans_segmentation_template.py`

### 商業解讀

- 定義每群客戶的價值、需求與對應行銷策略。

### 企劃落地句型

- 「分群結果顯示『高價值高風險流失群』占 18%，建議優先投入留存方案以提升年度 LTV。」

---

## 3) Chi-Square（類別關聯）

### 適用情境

- 檢驗兩個類別變數是否有關聯（如客群類型與購買與否）。

### 輸入欄位

- 類別欄位 `x_col`, `y_col`。

### 假設前提

- 觀測值獨立。
- 期望次數不宜過低（一般建議 >= 5）。

### Python 範例

- 參考 `scripts/chi_square_template.py`

### 商業解讀

- 驗證是否需為不同客群採差異化策略。

### 企劃落地句型

- 「卡方檢定顯示客群類型與續約行為顯著相關（p < 0.05），因此續約方案需分眾設計。」

---

## 4) One-way / Two-way ANOVA（群組差異）

### 適用情境

- 檢驗不同群組在某連續指標上的差異是否顯著。

### 輸入欄位

- `dependent`（連續變數）
- `group`（單因子）
- `second_group`（雙因子可選）

### 假設前提

- 殘差近似常態。
- 各組變異數具可比性。

### Python 範例

- 參考 `scripts/anova_template.py`

### 商業解讀

- 找出哪個群組或組合最具投入價值。

### 企劃落地句型

- 「雙因子 ANOVA 顯示『通路 x 價格方案』交互作用顯著，代表價格策略需依通路分開制定。」

---

## 5) Regression（連續目標預測）

### 適用情境

- 估計變數對營收、客單價、需求量等連續目標的影響。

### 輸入欄位

- `target`（連續）
- `features`（自變數）

### 假設前提

- 關係近似線性。
- 多重共線性可接受。

### Python 範例

- 參考 `scripts/regression_template.py`

### 商業解讀

- 量化每個槓桿對目標的邊際影響。

### 企劃落地句型

- 「回歸結果顯示提升產品啟用率每 1%，可帶動月營收增加約 2.3%，故優先資源放在啟用流程優化。」

---

## 6) Logistic Regression（二元結果機率）

### 適用情境

- 預測是否購買、是否流失、是否續約等二元事件。

### 輸入欄位

- `target`（0/1）
- `features`

### 假設前提

- Log-odds 與特徵近似線性。
- 樣本類別比例可用（必要時做重抽樣）。

### Python 範例

- 參考 `scripts/logistic_regression_template.py`

### 商業解讀

- 用勝算比（odds ratio）找出可操作的轉化槓桿。

### 企劃落地句型

- 「Logistic 模型顯示完成 3 個 onboarding 步驟的用戶，購買勝算提升 1.8 倍，應把 onboarding 完成率列為首要 KPI。」

---

## 7) Conjoint Analysis（屬性偏好權衡）

### 適用情境

- 產品、價格、方案組合取捨；找最佳 feature bundle。

### 輸入欄位

- 多個屬性欄位（類別）
- 偏好分數或選擇結果（`target`）

### 假設前提

- 問卷設計能反映真實取捨。
- 屬性與水準定義清楚。

### Python 範例

- 參考 `scripts/conjoint_template.py`

### 商業解讀

- 找出最受重視屬性與可接受價格區間。

### 企劃落地句型

- 「Conjoint 顯示『自動化報表』的效用值最高，且客戶對價格由 999 提升至 1299 的敏感度可接受，建議升級方案定價採 1299。」

---

## 8) Multi-Dimensional Scaling（定位圖）

### 適用情境

- 將品牌/產品在消費者心智中的相對位置視覺化。

### 輸入欄位

- `id_col`（品牌或產品識別）
- `feature` 欄位（可形成距離）

### 假設前提

- 距離度量可反映感知差異。
- 維度降到 2D 後仍保留足夠結構。

### Python 範例

- 參考 `scripts/mds_positioning_template.py`

### 商業解讀

- 定義與競品的接近程度與差異化方向。

### 企劃落地句型

- 「MDS 顯示我們目前與 A 品牌過度接近，建議在『服務深度』維度拉開定位並作為下一季主溝通軸。」
