# 準備階段：七要素盤點與 BATNA 工程

哈佛談判專案（Fisher & Ury）的七要素是準備階段的完整檢查清單。原則：**準備的品質決定談判上限，臨場技巧只決定你能拿到多少已準備好的東西。**

---

## 七要素逐項準備問題

### 1. Interests（利益）

立場是「他說要什麼」，利益是「他為什麼要」。每個立場至少往下挖一層。

我方：
- 這次談判我真正要解決的問題是什麼？排出優先序（不能只有一項「價格」）。
- 哪些利益是必須滿足的，哪些是有了更好的？
- 我的老闆／組織的利益和我個人的利益有沒有分歧？（例如：採購 KPI 是砍價幅度，但部門真正需要的是交期穩定）

對方：
- 對方檯面上的要求，背後可能對應哪 2-3 種利益？逐一列出並標可能性。
- 對方談判代表個人的利益是什麼？（升遷、KPI、不想擔責任、想快點結案）
- 對方組織此刻的壓力來源？（季底衝業績、庫存、預算凍結、上層指示）

### 2. Options（選項）

可以放上桌交換的所有議題，不只價格。

- 列出至少 5 個可交易變數：價格、數量、交期、付款條件、合約年限、服務範圍、獨家性、續約條款、風險分擔……
- 對每個變數標「對我的價值」與「推估對對方的成本」。**找出「對我價值高、對對方成本低」的項目**——這是打包交換的原料。
- 有沒有能把餅做大的選項？（例如長約換價、量增換折扣）

### 3. Alternatives（替代方案 / BATNA）

談不成時各自會怎樣。詳見下方 BATNA 工程一節。

### 4. Legitimacy（客觀標準）

- 我的開價和目標，分別能用哪些外部標準支撐？
- 對方可能引用哪些標準？我怎麼回應？
- 雙方標準衝突時，哪個標準的來源更中立、更近期、更貼近本案情境？

### 5. Communication（溝通）

- 用什麼管道談？（面對面＞視訊＞電話＞文字；越重要的議題用越高頻寬的管道）
- 誰上場？需不需要角色分工（主談／記錄／觀察）？
- 我計畫先問哪些問題？（準備 5 個以上開放式問題，先探後談）

### 6. Relationship（關係）

- 一次性還是長期？長期關係下，短期榨取的代價是什麼？
- 這場談判之後我們還要一起做什麼？（履約、續約、跨案合作）
- 關係議題和實質議題分開處理：不用讓利買關係，也不用破壞關係逼條件。

### 7. Commitment（承諾）

- 對面的人有沒有拍板權？沒有的話，真正的決策者是誰、怎麼觸及？
- 我自己的授權範圍到哪裡？超出範圍的要求怎麼回？
- 成交的承諾長什麼樣？（口頭→會議紀錄→合約，每一步的時程）

## 七要素填表模板

| 要素 | 我方 | 對方（推估） | 信心 |
|------|------|------------|------|
| Interests | 優先序 1/2/3 | 假設 1/2/3 + 依據 | 高/中/低 |
| Options | 可交易變數清單 + 各自價值 | 推估對方在意的變數 | 高/中/低 |
| Alternatives | BATNA 是什麼、強度 | 對方 BATNA 推估 | 高/中/低 |
| Legitimacy | 支撐我方的標準 + 出處 | 對方會拿什麼標準 | 高/中/低 |
| Communication | 管道 / 上場者 / 問題清單 | 對方談判者風格 | 高/中/低 |
| Relationship | 一次性或長期 / 修復需求 | 對方怎麼看這段關係 | 高/中/低 |
| Commitment | 我的授權範圍 | 對方拍板權 / 決策鏈 | 高/中/低 |

規則：對方欄位空白或全是「低信心」時，第一輪談判的目標應設為**蒐集情報**而非成交。

## BATNA 工程

### 步驟一：列出所有替代方案

談不成的話我能做什麼？窮舉，不篩選。常見類別：
- 換一個交易對象（別家供應商、別的 offer、別的房子）
- 自己做 / 內部解決
- 延後、暫時維持現狀
- 縮小範圍成交（只買一部分、先簽短約）
- 走正式程序（招標、升級到高層、法律途徑）

### 步驟二：可行性評估

對每個替代方案評三項：**可得性**（現在就有，還是要花時間爭取）、**價值**（換算成和本案同單位的價值）、**成本**（切換成本、時間、關係代價）。刪掉幻想型選項——「大不了我去創業」不是 BATNA。

### 步驟三：強化最佳者

BATNA 是拿來建設的，不是拿來發現的。挑出最佳替代方案後，在談判前投入行動讓它變得更真實：
- 薪資談判：真的去面試，拿到第二個 offer 或至少進入流程。
- 採購：真的向第二家供應商索取正式報價，而不是只查價目表。
- 租約：真的去看兩間備選物件並確認可簽。

**每強化一分 BATNA，桌上的底氣多一分，而且不需要說出口對方通常感覺得到。**

### Reservation Point 換算方法

Reservation point（底線）不是「我感覺最低能接受多少」，是從 BATNA 換算出來的：

```
reservation point = BATNA 的價值 ± 切換成本與風險調整
```

範例（薪資）：現職年薪 100 萬＋預期調薪 3%＝BATNA 約 103 萬；換工作的風險與適應成本估 5 萬 → 新 offer 的 reservation point ≈ 108 萬。低於這個數字，留在原地更划算。

範例（採購）：備選供應商報價 95 萬，但切換要重新驗證品質、估 8 萬一次性成本，攤提後本案等效 100 萬 → 現有供應商只要開價低於 100 萬就仍優於切換。

規則：
- Reservation point 只寫在自己的準備表上，**永遠不說出口、不暗示**。
- BATNA 改變時（新 offer 進來、備選供應商漲價）要立刻重算，不是談判開始後就凍結。

## 推估對方 BATNA

| 方法 | 具體做法 |
|------|---------|
| 公開資訊 | 財報、徵才頁（急徵＝人力缺口）、產能利用率、庫存水位、同業成交行情、法說會逐字稿 |
| 時間壓力訊號 | 誰先約會議、回覆速度、季底/年底節點、對方是否主動提 deadline。急的一方 BATNA 通常較弱 |
| 詢問測試 | 直接問開放式問題：「如果我們這次合作不成，你們接下來的規劃是？」對方回答的具體程度透露 BATNA 強弱——答得含糊，通常代表沒有好的替代 |
| 提案測試 | 丟一個略低於行情的試探條件，觀察對方是拒絕得果斷（有底氣）還是遲疑後討論（替代方案弱） |

## ZOPA 推估

```
我方 reservation point ─────┐
                            ├── 重疊區間 = ZOPA
對方 reservation point（推估）┘
```

- 我方底線已知，對方底線用上面的方法推一個區間（不是單點），例如「對方最高可能付 110-120 萬」。
- 推估的 ZOPA 決定 aspiration 怎麼設：目標應落在推估對方底線附近、由我方一側。
- 推不出 ZOPA 或推估區間不重疊：先別急著讓步，優先驗證推估（可能錯），或回頭做大餅（增加議題讓 ZOPA 出現）。真的沒有 ZOPA，執行 BATNA 就是正確結局，不是失敗。

## Aspiration Point 設定

研究結論一致：**設定具體且偏高目標的談判者，成果顯著優於「盡力就好」的談判者**（目標會錨定自己的讓步行為）。但有兩個約束：

1. **可辯護**：aspiration 必須有至少一個客觀標準能撐——被問「這個數字怎麼來的」時，答案不能是沉默。撐不住的高價會賠掉可信度，之後每個數字都被打折看待。
2. **和底線分開存放**：準備表上同時寫 aspiration（開場追求）與 reservation point（觸線走人），談判中盯著 aspiration 談，只有在對方逼近底線時才拿 reservation point 出來做決策，不拿它當談判目標。

## 客觀標準（Legitimacy）清單

依場景可用的標準來源：

- **市場行情**：薪資調查報告（104/1111 薪資情報、Glassdoor、獵頭數據）、同規格產品三家報價、實價登錄、租屋平台同區段行情
- **前例**：雙方過去成交條件、對方與同業的公開合作條件、公司內同職級敘薪帶
- **第三方鑑價**：公證行情、鑑價報告、產業公會定價指引、獨立顧問評估
- **成本結構**：原物料公開報價、匯率、運費指數（用於供應商漲價談判的合理性檢驗）
- **程序公平**：輪流出題、一方切一方選、共同委託第三方

使用規則：標準要在開價**同時**端出，不是被質疑後才補。「依照 X 調查，這個職級在本產業的 P75 是……所以我提出……」比先喊數字再找理由有力得多。
