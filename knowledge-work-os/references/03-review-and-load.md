# 回顧與負載管理：每週回顧、承諾預算、超載急救

系統靠回顧維持乾淨，靠負載管理避免超載。這兩件事沒做，01 和 02 建得再漂亮都會在三週內崩掉。

---

## 每週回顧完整腳本（30-60 分鐘）

固定時段執行（建議週五下午或週一早上，放進行事曆設提醒）。逐步照做，每步有明確完成標準：

### Step 1: 清空所有收件匣（10-15 分鐘）

- Email inbox、快速捕捉入口、桌上紙張、下載資料夾、瀏覽器待讀分頁
- 每條過 clarify 判斷樹（見 [01-gtd-engine.md](./01-gtd-engine.md)）：丟棄 / 歸檔 / 2 分鐘做掉 / 進清單
- 完成標準：所有收集點歸零。不是「處理完每件事」，是「每件事都有了去處」

### Step 2: 逐專案確認 next action 存在（10-15 分鐘）

- 打開 projects 清單，一條一條過
- 每個專案問：它現在在 next actions 或 waiting for 裡有對應項目嗎？
  - 沒有 → 當場補一條下一步行動（這是回顧最重要的產出）
  - 專案其實已完成 → 標完成、慶祝一秒、封存
  - 專案不會做了 → 移 someday 或刪除，誠實面對
- 完成標準：零停滯專案

### Step 3: 巡 waiting for 追蹤（5 分鐘）

- 逐條看：等多久了？超過預期日了嗎？
- 超期的 → 產生一條追蹤行動（「傳訊問 Amy 設計稿進度」）進 next actions
- 已回覆但你漏了的 → 當場處理或轉成行動

### Step 4: 掃 someday-maybe（3-5 分鐘）

- 快速掃過，不深想
- 有東西時機到了 → 升格為專案並定義第一步
- 看了三週都無感的 → 刪掉沒關係

### Step 5: 行事曆前瞻 2 週（5 分鐘）

- 往回看上週：有沒有會議產生了你還沒捕捉的行動或承諾
- 往前看兩週：即將到來的死線、需要提前準備的會議 → 產生準備行動
- 兩週而不是一週：一週的視野會讓「下週三的大簡報」在只剩兩天時才進入你的雷達

### Step 6: 定下週 big 3（5 分鐘）

- 綜合以上，選出下週最重要的三件事
- 檢驗：如果下週只完成這三件，這週算成功嗎？算，才是對的 big 3
- 為每件在行事曆上預佔至少一個深度區塊（見 [02-attention-design.md](./02-attention-design.md)）

### 縮水版腳本（15 分鐘，忙爆的那週用）

1. 清空快速捕捉入口與 email（5 分鐘，只分流不處理）
2. 只巡「本週動過的專案」確認下一步存在（4 分鐘）
3. waiting for 只看超期項目（2 分鐘）
4. 行事曆前瞻 1 週（2 分鐘）
5. 定下週 big 3（2 分鐘）

縮水版連續用兩週後，第三週必須跑完整版，否則清單開始腐敗。

### 錯過回顧的補救規則

- 錯過一次：下個工作日補跑縮水版即可，不要等下週——間隔拉到兩週，補跑成本會倍增
- 連續錯過兩次：檢查排程位置，通常是排在低精力或高衝突時段（週五 17:00 是最常見的錯誤位置）。換到受保護的時段，例如週一早上第一件事
- 回顧一直跑不完：第一次做完整回顧本來就慢（收件匣積壓），連續三週後會收斂到 30-45 分鐘。三週後還超過 60 分鐘，問題多半在 Step 1 積壓——每日維護儀式沒做，把每日的活堆給了每週

## 承諾預算管理

超載的根源不是效率不足，是**承諾進來的速度大於完成的速度**。效率優化解決不了結構性超收。

### 工時盤點

算出你真正的深度時數預算：

```
每週上班時數                          40
- 會議時數（看過去四週行事曆平均）      -12
- 例行事務（email、行政、每日維護）      -8
- 突發與救火（誠實估，通常 10-20%）      -5
= 可支配時數                           15
其中符合精力高峰、可用於深度工作的     ≈ 10-12
```

多數人第一次算完會震驚：自己以為有 40 小時，實際深度預算只有 10-15 小時。這就是為什麼「感覺整週都在忙但重要的事沒進度」——重要的事需要深度時數，而深度時數早就被超賣了。

### 新承諾進 = 舊承諾出

預算用滿之後（多數人長期處於用滿狀態），接新承諾的規則：

- 接下任何預估超過 2 小時的新承諾前，先問：**這要從哪個現有承諾手上把時間拿走？**
- 說得出來 → 接，並實際去重新協商那個舊承諾（延期、縮小、退出）
- 說不出來 → 這個「答應」就是對某個既有承諾的隱性違約，只是你還不知道是哪個
- 對主管的版本：「可以，我目前手上有 A、B、C。加上這件的話，你希望哪一個往後移？」——把取捨還給有權取捨的人，而不是自己吞下去然後全部延遲
- 已經在動用晚上與週末補進度的，不是忙，是結構性超收——直接跑下方超載急救，不要等爆

## 拒絕與談判模板

三種話術，都遵守同一結構：**明確的不 + 理由簡短 + 給對方一條路**。理由不要長篇解釋——解釋越長，越像在邀請對方反駁。

### 型一：拒絕但給替代

> 這件事我這邊排不進來，接下來三週我的重點在 X。你可以找 Amy 聊聊，這在她的守備範圍；或者如果能等到月底，我那時可以看一下。

### 型二：延期

> 我可以做，但最早要 [具體日期] 才能開始，因為手上 X 的死線在前面。這個時間對你來說行得通嗎？不行的話我們看看有沒有別的解法。

要點：給具體日期，不說「晚點」「盡量」。模糊的延期會被理解為「他答應了」。

### 型三：縮小範圍

> 完整版我沒辦法在你要的時間內給。但如果先給你 [縮小後的版本，例：只含 A 部分的初稿 / 一頁摘要 / 口頭結論]，能不能先解你的燃眉之急？完整版我 [日期] 給。

要點：縮小的版本要具體描述，讓對方能立刻評估夠不夠用。

## 超載急救 Triage

適用狀態：本週死線多到做不完、已經或即將開天窗、恐慌中。此時不要優化系統、不要學方法論，照做以下四步：

### Step 1: 全部倒出（15-20 分鐘）

所有承諾寫下來，不分大小：死線任務、答應別人的事、會議、例行工作。恐慌的很大一部分來自「不知道到底有多少」——Zeigarnik 機制在超載時火力全開，倒出來本身就能降壓。

### Step 2: 三分法（10 分鐘）

每條標一個字：

- **死**（本週死線）：不做會有真實後果（合約、客戶、法遵、公開承諾）。檢驗：後果具體說得出來嗎？說不出來的不算
- **延**（可延）：重要但死線可動，或死線是自己想像的
- **棄**（可棄）：做了很好但沒人會死，包括你自己答應自己的完美主義項目

誠實原則：第一輪分完「死」超過可支配時數的，回頭再砍一輪。全部都是「死」等於沒有 triage。

三分實例：

| 承諾 | 直覺分類 | 檢驗後 | 理由 |
|------|----------|--------|------|
| 週四給客戶的提案 | 死 | 死 | 合約時程，違約有真實後果 |
| 週五部門週報 | 死 | 延 | 死線是慣例不是承諾，跟主管說改週一沒人受傷 |
| 幫同事 review 簡報 | 死 | 延 | 對方的死線下週三，這週不做不會怎樣 |
| 內部工具優化 | 延 | 棄 | 自己起頭的項目，沒人在等，是完美主義項目 |
| 每週跨部門例會 | 死 | 延 | 請假一次，會議記錄補看 |

### Step 3: 主動通知，重新承諾（30 分鐘，最難也最關鍵）

對每個「延」和「棄」的利害關係人，**今天主動通知**，不要等對方來問：

> [名字]，跟你同步一下：我這週有 [X] 的死線壓著，原本說好的 [Y] 我沒辦法在 [原日期] 給你。我可以 [新日期] 給，或者先給你 [縮小版本]。哪個對你比較好？

主動通知 + 給新承諾 = 可靠的人遇到高峰；被動開天窗 + 事後道歉 = 不可靠的人。內容一樣，時機決定你是哪一種。

通知順序規則：

1. 先通知**後果最大**的（客戶、主管），不是最好開口的（好說話的同事）。人會本能先挑軟的講，把最難的拖到最後——那個才是真正會炸的
2. 涉及主管的取捨，用選擇題不用問答題：「A 和 B 我這週只做得完一個，你要哪個先？」
3. 通知走文字（email / 訊息）留下新承諾的記錄，口頭講完補一行文字確認

### Step 4: 砍掉最貴的低價值承諾（10 分鐘）

排完本週生存計畫後，看著「延」和「棄」清單問一題：**哪一條吃掉最多小時、但價值最低？** 那條不要延——直接退出。超載通常不是十件小事造成的，是一兩個「當初不該接、現在捨不得放」的大承諾造成的。退出一個大的，勝過擠壓十個小的。

### 急救後

度過本週後 48 小時內做一次完整每週回顧 + 工時盤點。超載會再發生的唯一原因：承諾預算的結構問題沒修，只把這次的火撲掉了。

## 精力管理

時數是容器，精力是內容物。同樣 2 小時，高峰與低谷的深度工作產出可差數倍。

- **找出高峰**：連續一至兩週，每天在 10:00 / 14:00 / 16:30 快速記錄精力（1-5 分）。多數人高峰在起床後 2-4 小時，午餐後有明顯低谷，但個體差異大——用記錄，不要用想像

```
精力記錄（每次 10 秒，設三個手機提醒）
日期    10:00   14:00   16:30   備註
一       4       2       3      午餐吃太飽
二       5       2       2      —
三       4       3       3      14:00 前散步 10 分鐘
→ 一週後看型態：這個人高峰明確在上午，14:00 是固定低谷
→ 排程結論：深度區塊排 9:00-12:00，例行會議爭取排 14:00-15:00
```
- **高峰時段留給深度工作**：這是行事曆防禦的優先保護區。高峰時段拿來開例行會或回信，是最貴的資源錯置
- **低谷排 shallow**：回信批次、行政、整理、例行會議，低谷做這些剛剛好，不需要罪惡感
- **睡眠是槓桿不是變數**：趕工砍睡眠，等於用明天的高峰時段換今晚的低谷時段——同樣的小時數，換到的是品質差數倍的認知。長期睡眠不足的人沒有高峰可言，所有精力管理技巧都失效。排程時把睡眠當固定成本，不當調節閥
