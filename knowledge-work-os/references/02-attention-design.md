# 注意力設計：時間箱、深度工作與干擾防禦

清單解決「做什麼」，這份文件解決「什麼時候做、怎麼保護做的品質」。

---

## Time Blocking 時間箱

原則：不是把待辦塞進行事曆，而是**替每一段工作時間預先指定用途**。沒被指定的時間會自動被最大聲的事情（訊息、臨時請託）吃掉。Parkinson's law 在這裡是工具：工作會膨脹到填滿可用時間，所以反過來用時間箱給工作設邊界，它就會收斂。

### 排程時機

前一天晚上或當天早上第一件事，擇一固定。前一天晚上排的額外好處：符合 Zeigarnik 機制，把明日計畫寫下後，晚上不會被「明天要幹嘛」的念頭糾纏。

### 排程步驟

1. 先抄下今日硬承諾（會議、死線）——這些是不可動的地形
2. 定今日 big 3：三件今天完成就算「今天有價值」的事，動詞開頭
3. 把 big 3 中需要深度工作的，排進精力高峰時段（見 03 精力管理）
4. shallow 工作（回信、行政、瑣事）打包成 1-2 個批次時段，塞低谷
5. 留 buffer（見下）

### 工作量預估 ×1.5 校正

人對自己任務的耗時預估系統性偏低（planning fallacy）。實務校正：**寫下直覺預估後乘以 1.5 再排箱**。覺得 2 小時的排 3 小時。連續兩週記錄「預估 vs 實際」，可以校出你個人的係數——多數人落在 1.3-2.0 之間。

校正記錄格式（每天排程時順手記，兩週即可收斂）：

```
任務              直覺預估   實際耗時   係數
寫季度簡報大綱      1h        1h50m     1.8
回覆客戶提案信      30m       45m       1.5
Code review #88    1h        1h10m     1.2
→ 兩週後取中位數當個人係數；「寫作類」和「例行類」可以分開算，通常寫作類係數更高
```

### Buffer 規則

- 每 2-3 個時間箱之間留 15-30 分鐘空白
- 每日總排程量不超過可支配時數的八成
- buffer 不是浪費，它吸收的是：會議延長、預估失準、突發插入。沒有 buffer 的時間箱表在第一個意外出現時整張作廢，然後你會得出「時間箱沒用」的錯誤結論

### 行事曆防禦

順序是關鍵：**先把深度時段放上行事曆佔住，再讓會議填剩下的空隙**。反過來（先讓會議自由預約、再找空檔做深度工作）的結果是深度時段永遠是碎片。

- 深度時段在行事曆上顯示為「忙碌」，名稱可以就叫工作區塊，不必解釋
- 每週至少防禦住 3-4 個 90 分鐘以上的完整區塊
- 有共享行事曆文化的公司，這招合法且有效；被質疑時的回答：「那時段我有既定工作承諾，X 時之後都可以」

### 範例日程（高峰在上午的典型知識工作者）

```
08:50-09:00  排今日時間箱（若前晚未排）
09:00-10:30  深度區塊 1：big 3 之一（高峰精力，零通知）
10:30-11:00  休息 + 批次 1：email 與訊息
11:00-12:00  深度區塊 2：big 3 之二（60 分鐘短區塊）
12:00-13:00  午餐（離開螢幕）
13:00-14:00  會議（協調型工作集中下午）
14:00-15:00  辦公時間：開放打斷、臨時討論、shallow 瑣事
15:00-15:30  批次 2：email 與訊息
15:30-16:30  彈性 buffer：吸收延長與突發；沒事就繼續 shallow
16:30-17:15  會議或協作
17:15-17:30  每日維護儀式 + 排明日（見 01）
```

重點不是照抄這張表，是結構：深度在高峰、批次固定、buffer 明排、維護收尾。

## 深度工作時段設計

### 時長：90-120 分鐘

Ultradian rhythm：人的警覺度以約 90-120 分鐘為週期起伏（源自 Kleitman 的 basic rest-activity cycle 研究），順著週期設計工作區塊，在自然低點休息，比硬撐更能維持整天的產出品質。

- 一個深度區塊 = 90-120 分鐘專注 + 15-20 分鐘真休息
- 真休息：離開螢幕、走動、發呆。滑手機不是休息，它是另一種注意力消耗

### 每日深度上限約 4 小時

Cal Newport 在 Deep Work 中整理的研究共識（含刻意練習研究的觀察）：**經過訓練的人，一天高強度深度工作的上限大約 4 小時**，新手更少。這是設計參數，不是勵志目標：

- 排出 5-6 小時深度時段的計畫必然失敗，然後產生罪惡感
- 正確目標：每天 2-4 小時真正的深度工作，其餘時間坦然做 shallow
- 一天 3 小時真深度，一週就是 15 小時，多數知識工作者實際深度時數遠低於此

### 新手起步

沒有深度工作習慣的人，專注肌力不足，直接排 90 分鐘會在第 25 分鐘開始摸手機。

- 從 60 分鐘起步，甚至 45 分鐘
- 連續一至兩週能穩定完成後，加到 75、再到 90
- 區塊內規則簡單化：只有一條——不碰這個任務以外的任何東西。卡住就盯著卡住的地方，不准「順便查個東西」

### 啟動儀式（降低進入摩擦）

深度區塊最大的損耗常在頭 10 分鐘的「進不去」。固定一套 2 分鐘啟動流程，讓開始變成自動動作：

```
1. 手機拿到視線外（另一個房間或抽屜，不是螢幕朝下）
2. 關閉所有無關分頁與通訊軟體，只留任務需要的
3. 讀上次的斷點筆記（見下方切換儀式），沒有就寫一行「這個區塊要產出什麼」
4. 計時器設定區塊長度，開始
```

區塊中途冒出的雜念（「要記得訂會議室」）：寫進快速捕捉入口一行，立刻回來。這正是捕捉系統對深度工作的支援——大腦相信寫下的不會丟，才肯放手。

## Attention Residue 注意力殘留

Sophie Leroy（2009）的研究發現：從任務 A 切到任務 B 時，注意力有一部分仍殘留在 A 上，導致 B 的表現下降。殘留在兩種情況下最嚴重：**A 未完成就切走**、以及**切換時沒有時間壓力**。也就是說，「快速看一眼訊息再回來」不是免費的——付出的是回來之後好幾分鐘的降級認知。

### 對策一：減少切換次數

批次處理同類任務（見下方干擾管理），深度區塊內零切換。

### 對策二：切換儀式（不得不切時）

被迫中斷或到達區塊結尾但任務未完時，離開前花 60-90 秒寫下：

```
【斷點筆記】
- 現在做到哪：（例：報告第二節寫到成本分析，缺 Q2 數據）
- 下一步是什麼：（例：跟財務要 Q2 數據，拿到後補進第三段）
- 剛剛在想什麼：（例：考慮要不要把風險段落移到最前面）
```

這利用的是 Masicampo & Baumeister 的機制：寫下具體計畫讓大腦「放下」未完成任務，減少殘留；同時下次接手時省掉 10-15 分鐘的「我剛剛做到哪」重建成本。

## Implementation Intentions（if-then 計畫）

Gollwitzer 提出的執行意圖：把「我要做 X」改寫成「**if 情境 Y 出現, then 我就做 X**」。Gollwitzer & Sheeran（2006）的統合分析涵蓋 94 個獨立測試，效果量 d ≈ 0.65，屬中到大效果——在行為改變研究裡是相當強的結果。

### 為何比單純意圖有效

單純意圖（「我要多寫報告」）把行動的啟動交給當下的意志力與判斷。if-then 把啟動綁定到具體情境線索上，情境出現時行為近乎自動觸發，**跳過了當下決策**——而當下決策正是拖延發生的地方。

### 格式規則

- if 部分必須是**可辨識的具體情境**：時間+地點、或明確事件。「有空的時候」不合格
- then 部分必須是**單一具體動作**，動詞開頭

### 兩種用法

**啟動型**（讓該做的事自動開始）：

```
if 週二 9:00, then 進會議室 B 寫報告第一節
if 站會結束回到座位, then 先開 IDE 處理 #142 這張票，不開信箱
if 週五 16:00 行事曆提醒跳出, then 開始每週回顧
```

**攔截型**（阻斷壞習慣的自動迴路）：

```
if 出現打開社群的衝動, then 站起來喝一杯水
if 想「先看一下訊息」, then 在斷點筆記寫一行再說
if 有人走過來說「借一分鐘」而我在深度區塊, then 回「我 11 點過去找你」
```

每天的 if-then 計畫 2-3 條就夠。針對「今天最可能出事的地方」設計，不要寫十條——寫十條等於零條。

## 干擾管理

### Email 與訊息批次

- 收信/收訊息固定 2-3 個時段（例：10:30、14:00、16:30），每次 20-30 分鐘處理完畢
- 批次時段外關閉信箱分頁與通知
- 心理帳要算清楚：即時回覆的價值是「別人少等 1 小時」，代價是「你整天沒有完整的 30 分鐘」。多數職位這筆帳是虧的
- 真正的緊急事不會走 email——會打電話、會走過來。用這句話回應「萬一有急事」的焦慮

### 通知白名單

預設全關，再開白名單：

- 手機：只留電話、行事曆提醒、與 1-3 個真緊急管道（例：主管直訊、on-call 告警）
- 電腦：關掉所有 badge 和橫幅，包括 email 圖示上的未讀數字
- 群組訊息一律不即時通知，批次時段看

### 辦公時間制度

與其被隨機打斷十次，不如公告固定的可打斷時段：「每天 14:00-15:00 歡迎來找我討論任何事，其他時間請留言」。給同事一個確定的入口，換取其餘時間的完整性。主管尤其適用——這同時是給團隊的預期管理。

## 會議防禦

### 預設 25 / 50 分鐘

把行事曆預設會議長度從 30/60 改成 25/50。Parkinson's law 的直接應用：給 60 分鐘的會就會開滿 60 分鐘。截短的 5-10 分鐘是換場 buffer，避免連續會議的 attention residue 疊加。

### 無議程不接受

收到沒有議程的會議邀請，回覆模板：

> 想先確認這場會要決定或討論什麼，方便我準備。可以補一下議程或一句話目標嗎？

三種結果都是賺：對方補了議程（會議品質上升）、對方發現用一封信就能講完（會議消失）、對方撤回邀請（本來就不需要你）。

### 非同步取代判斷

收到會議邀請時過這個判斷：

```
這場會的目的是什麼？
├─ 單向傳達資訊 → 一封信或一份文件即可，建議取代
├─ 收集大家意見 → 文件留言收集，只有意見衝突時才需要會
├─ 做一個決定 → 決策者 + 必要輸入者的小會（≤4 人），你不是其中之一就婉拒
├─ 腦力激盪 / 對齊複雜歧見 / 敏感人事 → 真的需要同步會議
└─ 例行同步會 → 問：上三次這個會，有產生過任何決定或行動嗎？沒有就提議改雙週或改文字更新
```

婉拒話術見 [03-review-and-load.md](./03-review-and-load.md) 的拒絕模板。
