# 定位理論操作手冊（Ries & Trout + Positioning Statement）

## 定位聲明公式

```
For [target] who [need], [brand] is the [category] that [POD], because [RTB].
針對 [目標客群]，他們 [未被滿足的需求]，[品牌] 是 [品類框架] 中 [關鍵差異點] 的選擇，因為 [信任理由]。
```

### 逐欄位填寫規則

| 欄位 | 填寫規則 | 常見錯誤 |
|---|---|---|
| target | 用「行為或情境」切分，具體到能想像出一個人。要能回答「誰不是我們的客群」 | 人口統計拼盤（「25-45 歲都會女性」），或「所有人都是客群」 |
| need | 用顧客自己的語言描述張力，來自訪談或評論原句，不是品牌的想像 | 把品牌想賣的功能反寫成需求（「需要 AI 排程功能」——沒有顧客這樣說話） |
| category | 決定顧客拿你跟誰比。品類一換，競爭對手、價格帶、評價標準全換 | 品類太寬（「生活風格品牌」等於跟所有人競爭）；自創品類但沒人聽得懂 |
| POD | 單一最強差異點，不是三個並列。多個 POD = 沒有 POD | 列了三個「特色」，每個都平庸；或選了對手也能宣稱的點 |
| RTB (reason to believe) | 可驗證的具體理由：技術、專利、獨家資產、流程、出身、實測數據 | 用形容詞當理由（「因為我們最用心」）；RTB 無法被第三方查證 |

填寫順序建議：category → target → need → POD → RTB。先定品類框架，其他欄位才有比較基準。

### 定位聲明不是 slogan

| | 定位聲明 | slogan / tagline |
|---|---|---|
| 讀者 | 內部團隊與代理商 | 消費者 |
| 功能 | 對齊產品、定價、通路、溝通的所有決策 | 記憶點與情緒 |
| 標準 | 精準、可檢驗、無歧義 | 好記、有感染力 |
| 壽命 | 五年以上 | 可隨 campaign 更換 |

把定位聲明直接拿去當廣告詞，或把想到的 slogan 反填回定位聲明，都是流程倒置。順序永遠是：定位聲明 → 訊息策略 → slogan。

## 品類框架選擇：進既有品類 vs 創新品類

預設進既有品類。創新品類需「同時」滿足三條件才考慮：

| 判斷面向 | 進既有品類 | 創新品類 |
|---|---|---|
| 心智現況 | 品類認知已建立，顧客知道怎麼比較 | 現有品類第一名地位穩固，正面攻擊無勝算 |
| 產品本質 | 產品是既有品類的更好版本 | 產品確實跨界，硬塞進舊品類會被錯誤比較 |
| 資源 | 教育成本低，行銷花在差異點 | 有資源做品類教育（第一年預算主要花在「解釋這是什麼」，不是「為什麼選我」） |

- 創新品類的隱藏成本：你教育出來的品類，追隨者可以免費搭便車。沒有防禦資產（通路、專利、網路效應）就別開品類。
- 中間路線：子品類（subcategory）。在既有品類內切出新分支（「氣泡水」之於「瓶裝水」），教育成本低於全新品類，又能避開正面競爭。多數「重定位」案例的最佳解在這層。

## POP / POD

### 先滿足 Points of Parity，才有資格談 Difference

POP 兩類：

- **category POP**：品類入場券。顧客預設你「本來就該有」的基本項。缺任何一項，顧客直接淘汰你，根本輪不到比較 POD。
- **competitive POP**：抵銷對手 POD 的項目。不用贏，做到「夠好、不構成拒買理由」即可。

檢核問題：「顧客會不會因為我們缺某個基本項，連比較都不比較就走掉？」有，就先補 POP，暫緩差異化溝通。

### POD 三準則（三缺一即不合格）

| 準則 | 判斷問題 | 不合格訊號 |
|---|---|---|
| desirable 顧客在乎 | 這個差異影響顧客的選擇或付費意願嗎 | 訪談中不提示就沒人提到；只有內部人覺得重要 |
| deliverable 做得到 | 現在做得到，且能持續交付嗎 | 靠補貼或硬撐才成立；一線員工做不出承諾的水準 |
| differentiating 有區隔 | 對手能不能明天就宣稱一樣的話 | 差異點是整個品類的通用形容詞（「高品質」「用心服務」） |

操作順序：先列 POP 清單確認達標 → 從剩餘資產中挑 POD 候選 → 逐一過三準則 → 存活者進定位聲明。

## 定位測試四關

定位聲明草案必須逐關通過。任一關不過，標明並回頭修改，不得帶病輸出。

| 關卡 | 測試方法 | 不過時的處置 |
|---|---|---|
| 1. 顧客在乎嗎 | 不提示訪談中顧客是否主動提到此需求；此差異是否造成付費意願差距 | 換 POD，或 ladder up 到顧客真正在乎的利益層 |
| 2. 做得到嗎 | RTB 可否被第三方驗證；營運端能否穩定交付；一線員工聽得懂並做得到嗎 | 縮小承諾範圍，或先補營運能力再溝通 |
| 3. 對手難抄嗎 | 對手複製需要多久、多少成本；差異是否綁定品牌獨有資產（專利、出身、通路、社群） | 把 POD 從「功能宣稱」改綁到獨有資產或品牌敘事 |
| 4. 能撐五年嗎 | 是否綁死在會過時的技術規格或短期趨勢；抽象層級夠不夠容納產品線演化 | ladder up 一層；把規格從 POD 降級為 RTB |

## Laddering：調整抽象層級

梯子四層：屬性（attribute）→ 功能利益（functional benefit）→ 情感利益（emotional benefit）→ 價值（value）。

- **ladder up 時機**：POD 被抄了、過不了「撐五年」關、品牌要擴產品線。風險：升太高變空話，「讓生活更美好」任何品牌都能講——升一層就停，並保留下層 RTB 支撐。
- **ladder down 時機**：新品牌無認知、現有主張太抽象沒人記得、需要立即轉換。風險：綁死在單一屬性，屬性一過時定位跟著報廢。
- 配置規則：POD 通常落在功能利益或情感利益層；RTB 落在屬性層。「屬性當證據、利益當主張」是最穩的組合。

梯子示範（吸塵器）：

```
屬性「無線 + 800g」→ 功能利益「隨手就能吸」→ 情感利益「家裡隨時見得了人」→ 價值「從容的生活」
定位落點：POD 取「隨手就能吸」（功能利益層），RTB 取「無線 + 800g」（屬性層）。
「從容的生活」不進定位聲明——任何品牌都能講，留給品牌敘事用。
```

## 重定位風險檢查

重定位前必跑，輸出進 `migration_risk`：

1. **相容性矩陣**：列出現有顧客的購買理由，逐一標記與新定位的關係——相容 / 中立 / 衝突。
2. **流失量化**：衝突客群佔現有營收的比例。**判斷規則：衝突客群營收占比超過三成，優先考慮副品牌或子品牌承接新定位，不動主品牌。**
3. **連動檢查**：新定位是否需要同步變動價格帶、通路、產品線？只改溝通不改體驗的重定位，會製造 identity 與 image 的新落差（見 02 參考文件）。
4. **能力檢查**：內部能力支撐得起新 RTB 嗎？定位喊出去、營運跟不上，比不重定位更傷。
5. **過渡設計**：雙軌溝通期多長、對既有顧客怎麼說（重點：讓老顧客覺得「品牌升級了」而不是「品牌不要我了」）。

## 產出流程 Checklist

按序執行，每步有產出物才進下一步：

1. 蒐集輸入：產品事實清單、競品定位主張、顧客語言樣本（訪談 / 評論原句）。
2. 品類框架決策：預設既有品類；檢查創新品類三條件；評估子品類路線。記錄決策理由。
3. POP 盤點：列 category POP 與 competitive POP，逐項標「達標 / 未達標」。有未達標項 → 標示為前置工作。
4. POD 候選：從產品事實與獨有資產列 3-5 個候選，逐一過三準則，做成淘汰表。
5. 填公式：按 category → target → need → POD → RTB 順序填。
6. 四關測試：逐關記錄過 / 不過與證據。不過的關卡回到對應步驟修改。
7. 重定位案加跑風險檢查五項。
8. 輸出：定位聲明 + POP/POD 表 + 四關結果 + 淘汰理由（`alternatives_rejected` 讓未來的人知道當初為什麼不選那條路）。

## 完整中文範例

### 範例一：B2B SaaS（微型企業會計工具）

> 針對「沒有會計背景、自己記帳」的 10 人以下小店家，他們最怕報稅期手忙腳亂，「帳點通」是雲端記帳工具中「把發票拍照就自動完成分錄與申報草稿」的選擇，因為內建台灣稅制規則引擎並與財政部電子發票平台直連。

- target 用行為切（自己記帳、無會計背景），不用產業切。
- category 選「雲端記帳工具」既有品類，不自創「智慧財稅平台」——顧客聽不懂。

| 關卡 | 結果 | 證據 / 條件 |
|---|---|---|
| 顧客在乎嗎 | 過 | 訪談中「報稅焦慮」不提示提及率高 |
| 做得到嗎 | 過 | 規則引擎已上線，客服可支撐申報季量能 |
| 對手難抄嗎 | 過 | 電子發票平台直連需申請資格 + 台灣稅制 know-how 累積 |
| 能撐五年嗎 | 條件過 | 稅制長存；「拍照」若被 OCR 普及化，需 ladder up 到「申報零焦慮」 |

### 範例二：消費品（機能燕麥飲）

> 針對「想喝拿鐵但乳糖不耐」的上班族，他們不想在口感和腸胃之間二選一，「穀感」是植物奶中「專為咖啡設計、拉花起泡不輸鮮奶」的選擇，因為採用咖啡師共同開發的雙倍燕麥蛋白配方。

- 品類選「植物奶」子品類切入（咖啡場景），避開與燕麥奶第一品牌正面比「健康」。
- POD 綁定使用場景（咖啡），differentiating 靠場景聚焦而非成分獨佔。
- 淘汰的替代方向：「最健康的植物奶」——不過第三關，健康是品類通用主張，且第一品牌已佔據。
- 風險標註：若對手推出咖啡專用款，POD 需 ladder up 到「咖啡師背書的專業感」，RTB 轉向合作咖啡店網絡。

### 範例三：服務業（寵物美容）

> 針對「把狗當家人、擔心送洗過程看不到」的飼主，他們要的不是洗乾淨而是放心，「毛好在」是寵物美容中「全程直播、不關籠等待」的選擇，因為每個美容檯配獨立鏡頭，且採預約制單獨服務動線。

- need 從「洗乾淨」ladder up 到「放心」——訪談中飼主抱怨的都是不透明，不是清潔品質。
- POD「全程直播 + 不關籠」可被抄，但綁定「單獨服務動線」的營運設計，抄襲需要重改坪效模型，成本高。
- 四關的「做得到」是關鍵風險：直播對美容師是壓力，需先確認一線接受度再對外承諾。
- POP 提醒：定位主打「放心」之前，「洗得乾淨」這個 category POP 必須先無爭議。清潔品質有客訴時，直播反而放大問題。

## 本檔案判斷的失效條件

- 「預設進既有品類」在平台型 / 網路效應產品上可能失效——先佔品類定義權的價值可能高於教育成本。
- 「衝突客群占比超過三成就用副品牌」是經驗閾值，資本充足、原品類正在崩塌時，果斷整體重定位可能更對。
- 範例中的四關證據是示意格式，實際使用時必須以真實訪談與數據代入，不得沿用範例的結論。
