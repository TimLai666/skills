# SCCT 情境危機溝通操作手冊（Coombs）

核心邏輯：回應策略不是看「事情多嚴重」選，而是看「公眾把多少責任歸給品牌」選。順序固定：先歸因分類 → 再修正因子調整 → 才選回應策略。跳過前兩步直接寫聲明，是最常見也最貴的錯誤。

## Step 1：危機責任歸因分類

| Cluster | 危機類型 | 公眾歸因的品牌責任 |
|---|---|---|
| victim 受害者群 | 天災、謠言與不實指控、產品被惡意竄改、外部攻擊（駭客）、職場暴力（依 SCCT 原典含員工個人行為之攻擊，組織自身亦為受害方） | 低——品牌自己也是受害者 |
| accidental 意外群 | 技術故障導致的事故或召回、無意的作業疏失、利害關係人挑戰（被指控但事實有爭議） | 中——非蓄意但發生在你場子 |
| preventable 可預防群 | 人為過失導致事故、明知風險仍為之、違規違法、管理失當、高層不當言行、數據造假 | 高——公眾認為你本可避免 |

判斷規則：

- 用「公眾視角」不是法務視角。判斷問題：「一般人會不會覺得這件事品牌本來可以避免？」法律上無責不等於歸因上無責。
- 混合型事件（外部攻擊 + 內部防護鬆散）：以公眾感知最重的成分為準，寧可高估責任也不要低估——低估的代價是回應不足被輿論二次審判。
- 分類會隨事實揭露移動。首發時是 accidental，查出內部早有警訊就變 preventable，回應策略要跟著升級，不能停在第一版。

## Step 2：修正因子

初步分類後，用兩個因子調整責任等級：

| 修正因子 | 判斷 | 調整 |
|---|---|---|
| 危機歷史 | 同類事件過去發生過嗎（含業界皆知的未爆彈） | 有 → 責任等級上調一級：victim 視同 accidental、accidental 視同 preventable |
| 既往聲譽 | 品牌先前爭議多、負面聲量高 | 差 → 上調一級；好 → 是緩衝但「不減級」（好聲譽讓公眾願意聽你說，不代表原諒） |

魔鬼氈效應（velcro effect）：有危機歷史的品牌像魔鬼氈，新危機的責任會黏得又快又牢。這也是危機後「防再犯」比「道歉」重要的原因——它決定下一次危機的起始責任等級。

## Step 3：回應策略矩陣

四個策略群，由弱到強：

| 策略群 | 具體手段 | 適用情境 | 誤用風險 |
|---|---|---|---|
| deny 否認 | 反擊指控者、直接否認、切割代罪者 | 純謠言且握有可公開的反證 | 事後被證實有責 → 毀滅性。說謊比原始危機傷得更重 |
| diminish 淡化 | 找理由（非我能控制）、辯解（傷害沒那麼大） | accidental 且無危機歷史 | 對受害者使用顯得諉過卸責；「傷害沒那麼大」由受害者定義，不由品牌定義 |
| rebuild 重建 | 補償、完整道歉 | preventable 一律；accidental + 有危機歷史 | 過早全面道歉於 victim cluster = 自認全責，徒增法律風險與責任感知 |
| bolster 增強 | 提醒過往善行、感謝支持者、自陳也是受害者 | 任何 cluster 的「配菜」 | 單獨當主菜 = 被讀成轉移焦點，火上加油 |

選擇規則：

- victim（修正後仍為 victim）→ 以資訊提供 + 關懷為主，可搭 bolster；不主動道歉，但表達對受影響者的關心。
- accidental → diminish + 矯正行動；有危機歷史則直接升 rebuild。
- preventable → rebuild，無例外。**preventable 用 deny = 火上加油**，每一次否認都在為後續反轉加碼利息。
- 策略可疊加但不可矛盾：邊否認邊補償，公眾只會記得「心虛」。

## 道歉有效三要素

rebuild 的道歉缺一即被讀成公關操作：

1. **承認具體錯誤**：指名做錯什麼。「我們在 X 環節未執行 Y 檢核」，不是「造成不便深感抱歉」。
2. **受害者補救**：具體、主動、成本可見。受害者不用自己申請、舉證到死才拿得到。
3. **防再犯改變**：制度性改變 + 時間表 + 後續公開進度。「加強教育訓練」不算，那是每次都講的空話。

假道歉語言黑名單（出現即重寫）：

- 「如果有人因此感到不舒服」（把錯誤轉成受眾的感受問題）
- 「造成困擾，敬請見諒」（規避錯誤本體）
- 被動句式「錯誤被發生了」（沒有行為者）
- 「我們也是受害者」用在 preventable cluster（諉過）

## Stealing Thunder 自曝原則

研究結論：組織「自己先揭露」危機，聲譽損害顯著低於被第三方爆料。機制：自曝讓事件顯得較不嚴重（「敢自己講表示可控」）、剝奪媒體的獨家新聞性、拿回敘事框架的定義權。

適用判斷三問：

1. 外部是否已有人掌握此事（記者、離職員工、受害者、監管機關）？
2. 法規是否要求揭露（個資外洩通報義務等）？
3. 隱瞞被抓包的機率 × 代價，是否大於自曝的即期成本？

任一為「是」→ 自曝。三者皆否且已內部矯正 → 可不主動公開，但完整留存處理紀錄，為未來可能的揭露準備時間軸證據。

注意：自曝不等於自首式全認。自曝的內容仍按歸因分類配策略，victim cluster 的自曝重點是「我們發現了問題、正在處理」，不是道歉。

## 黃金時間與首發聲明

- 社群時代的黃金時間是「1 小時內有初步回應」，傳統的 24 小時規則已失效。輿論真空每多一小時，就由猜測、截圖和對手填滿。
- 首發不求完整，求「佔位」：宣示品牌已知情、在處理、可信賴。事實不齊全不是沉默的理由。

### 首發聲明四段式模板

```
【已知】我們已確認：（只寫查證過的事實，一到三點）
【未知】以下事項仍在查證中：（明列，不猜測、不硬答）
【正在做】我們已採取：（已啟動的具體行動：停售、通知、調查、配合機關）
【何時更新】我們將於（具體時間）提供下一次更新，無論屆時是否有新進展。
```

填寫規則：

- 不承諾原因（原因常在後續調查中反轉）、不推測、不指責任何一方、不辯解。首發裡的每一句辯解都是後續的引火點。
- 「何時更新」必須是具體時間並且做到。到點沒新事實也要發「目前無新進展、下次更新時間」——守時本身就是可信賴的證明。
- 有受害者疑慮的事件，首段前加一句對人的關心，先於對事的說明。

### 首發聲明填寫範例（食品品牌，顧客貼文稱產品中有異物）

> 我們已看到今日關於本公司 X 產品的貼文，對這位顧客的經歷感到不安，已於第一時間主動聯繫本人。
> 【已知】該批號產品於 O 月 O 日產出，我們已調出該批完整生產與檢驗紀錄。
> 【未知】異物來源仍在查證，不排除任何可能，包括製程、物流與其他環節。
> 【正在做】該批號產品已全面暫停出貨；已送第三方檢驗；已與顧客聯繫處理。
> 【何時更新】明日 12:00 前公布初步查證結果，無論屆時是否有新進展。

示範重點：沒有說「我們的製程不可能出問題」（辯解）、沒有說「疑似顧客保存不當」（指責）、暫停出貨的成本展示了認真程度。

## 完整案例走一遍（三步驟示範）

情境：外送平台深夜大規模當機，訂單消失，商家與消費者雙邊炸鍋。半年前發生過一次類似當機。

1. **歸因分類**：技術故障 → accidental cluster（中責任）。
2. **修正因子**：半年前有同類事件 → 危機歷史成立，責任上調一級，視同 preventable。
3. **策略選擇**：rebuild——具體補償（受影響訂單退款 + 商家損失補貼）+ 道歉三要素（承認「上次事件後的系統冗餘改善未完成」這個具體錯誤 + 補償方案 + 附時間表的架構改善計畫與公開進度頁）。
4. **對照錯誤做法**：若照初判 accidental 用 diminish（「流量異常非戰之罪」），輿論立刻翻出半年前的當機打臉，品牌信用二次受損——這就是跳過 Step 2 的代價。

## 社群時代升級判斷

回應層級階梯：個別留言回覆 → 置頂統一回覆 → 官方正式聲明 → 高層 / CEO 出面。

監測基準：平時就記錄「日常負面聲量基準值」與「品牌名 + 負面詞」的搜尋組合。危機判斷需要的是倍數與斜率（幾倍於日常、還在升或已見頂），沒有平時基準就無從判斷升級時機。

| 升級 | 觸發條件（任一即升） |
|---|---|
| 回覆 → 官方聲明 | 跨平台擴散（不只原發文平台）；媒體來詢問；KOL 或大帳號轉發；出現整理串或 hashtag；負面聲量達日常基準的數倍且連續上升 |
| 聲明 → CEO 出面 | 有人員傷亡或重大健康危害；大規模個資或資料外洩；企業誠信本體被質疑（造假、隱瞞）；中層與發言人回應已失效、輿論點名要負責人 |

CEO 出面的使用規則：CEO 是最後一張牌——一旦出面就沒有更高層級可再升，失敗了無人能救。非上述觸發條件時讓發言體系運作，CEO 留給不可替代的時刻。

危機期間內部作業規則：

- 對外發言單一窗口，所有部門的對外回應（含個人社群帳號被詢問時）一律導回窗口。
- 社群留言區不刪負評（截圖時代，刪文本身會變成第二個危機），只刪違法與人身攻擊內容，並公告刪文標準。
- 排程貼文全面暫停。危機聲明下面接著跳出促銷排程文，是最常見的自傷。
- 內部同步先於外部發布：員工從新聞得知自家危機，等於製造數百個失控發言點。

## 危機後品牌修復路徑

危機主要打擊 CBBE 的 judgments 層（credibility / trust），修復路徑對接 02 參考文件：

1. **承諾兌現期**（危機收尾～數月）：公開回報防再犯措施的執行進度。做了什麼、進度多少、第三方驗證了沒。此期的每一次守信都在降低下次危機的起始責任等級。
2. **靜默修復期**：不急著行銷洗版。用產品與服務的實際體驗累積正向證據，讓顧客「自己發現」品牌變了，比品牌自己說有效。
3. **重新敘事期**：時機判斷——事件搜尋聲量回落至基準、負面聯想在調查中不再前三名浮現。此時才把改變寫進品牌敘事（「那次事件讓我們重建了整個 X 流程」）。
4. **禁忌**：危機平息後短期內不用危機事件玩自嘲行銷梗。自嘲的前提是公眾已經原諒，順序不能反過來。是否已原諒，用調查與輿情驗證，不用自我感覺判斷。

## 本檔案判斷的失效條件

- SCCT 的歸因框架以「聲譽保護」為目標。涉及人身安全的事件，安全行動永遠先於聲譽計算——先召回、先停用、先通報，再談溝通策略。
- 「1 小時黃金時間」適用於已在社群擴散的事件；未擴散的內部發現事件，適用 stealing thunder 判斷，不是搶快。
- 回應策略矩陣假設公眾是主要利害關係人。B2B 品牌的危機常是「三個大客戶」而非大眾輿論，此時一對一溝通優先於公開聲明。
- 法遵要求（如個資外洩通報、重大訊息揭露）高於本手冊的任何策略建議，衝突時以法遵為準並讓法務參與每一版聲明。
