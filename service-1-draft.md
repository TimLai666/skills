服務設計實踐指南：以「提前配餐服務」掌握兩大核心工具

1. 導讀：從使用者到提供者的視角轉換

歡迎來到服務設計的深度實踐課。在之前的學習中，我們透過「人物誌」與「服務旅程地圖」學會了如何感同身受使用者的喜怒哀樂。然而，要讓一個美妙的體驗真正落地，我們必須完成一次關鍵的視角轉換：從「使用者」轉向「服務提供者」，看見那套支撐體驗運行的複雜機器。

服務設計的核心精神在於：

「讓所有元素在對的時間做對的事。」

本指南將帶領你掌握兩大核心工具——「生態系地圖」與「服務藍圖」。我們將先退後一步，看見服務所處的宏觀大環境，再深入細節，拆解運作流程。準備好從一位體驗者，進化為整場服務的「建築師」了嗎？

2. 生態系地圖 (Ecosystem Map)：掌握宏觀的服務全景

2.1 什麼是生態系地圖？

生態系地圖是以宏觀角度展示在大環境中，成就一個服務所涉及的「人」與「非人類元素」之間的關聯。它將服務視為一個系統，展示整體是如何連結的。

這項工具最適合應用於以下兩大時機：

1. 複雜社會議題： 如原住民正義、生態保育或動物保育，幫助我們看見隱藏的利害關係。
2. 盤根錯節的系統： 如醫療系統，讓開發者能理清多方交織的脈絡。

2.2 核心元素實踐：以「提前配餐服務」為例

在「提前配餐服務」中，使用者選取菜單，系統準備食材與食譜並送貨到府。要繪製這套服務的生態系，我們必須包含三大核心元素：參與者、關係與價值交換、以及區塊。

參與者 (Actors)

在生態系中，參與者可以是人、組織、地點或技術系統。

參與者類型	具體範例
個人 / 職位	餐點訂購者、派送員、食材供應商人員、客服人員
組織 / 部門 / 地點	食物銀行、品牌倉儲（地點）、研發部門、行銷部門、貨運公司
系統 / 平台	訂購 APP、金流系統、社群媒體廣告演算法、E-mail 訂閱系統

關係與價值交換

參與者之間不只有人的互動，還包含人機互動、機器對機器 (M2M)、系統對系統的深度連結。交換的價值主要分為三類：

* 資訊與數據： 使用者在社群媒體分享的個人資料、客服詢問的過敏源數據。
* 貨品與食材： 供應商提供的生鮮、包裝材料、最終送達的餐點。
* 金錢與勞力： 訂購者支付的費用、派送員提供的勞務。

2.3 關鍵技巧：設定「中心位 (C位)」與劃分區塊

地圖的視角取決於誰站在中心。「中心位一旦改變，整個生態系的視角也會隨之翻轉。」

* 視角轉向思考： 若以「餐點訂購者」為中心，我們會關注他如何接觸到廣告與訂購介面；但若將「派送員」換至 C 位，地圖將轉而呈現物流路徑、倉儲地點與派送系統的關聯。
* 劃分區塊 (Sectoring)：
  * 同心圓法： 依接觸強度劃分（直接接觸、間接接觸、無接觸）。
  * 角色功能法： 將地圖劃分為「供應鏈」（左上）、「政策與機構」（左下）、「產品部門」（右上）及「行銷部門」（右下）。

看清了誰在場之後，下一步我們要拆解這些參與者是如何在時間軸上互動的。

3. 服務藍圖 (Service Blueprint)：透視前台與幕後的運作流程

3.1 服務藍圖的定義與價值

服務藍圖以服務提供者的視角，展示前台與後台元素如何串聯。它有兩大核心應用價值：

1. 改善服務效率： 診斷流程中的不流暢之處。
2. 提升內部員工體驗： 當前端體驗已完善，轉而優化內部員工的操作流程與支持系統。

3.2 三條關鍵基準線：劃分可見度與互動

在繪製藍圖時，最上層應直接導入你的「使用者旅程」與「接觸點」。接著利用三條線區分層次：

1. 互動線 (Line of Interaction)

* 之上： 使用者的活動（如：註冊付款、簽收快遞）。
* 之下： 前台工作。在此建議加入一個重要的子類別——「科技層面 (Tech Layer)」。
  * 科技互動： APP 介面、Email 折扣碼自動折抵、網站訂餐表單。
  * 真人互動： 宅配員送餐時的簽收動作、發送實體傳單。

2. 可見線 (Line of Visibility)

* 之上： 使用者看得見的前台服務。
* 之下： 幕後工作 (Backstage)。使用者看不見但必須存在的流程，例如：後台更新菜單、接收訂單後訂購食材、倉儲人員裝箱。

3. 內部互動線 (Line of Internal Interaction)

* 之上： 服務團隊的內部活動（如：研發新菜品、供應鏈管理部門審核供應商）。
* 之下： 外部支持系統與第三方機構。例如：金流支付串聯、第三方電子報系統、或者是與食物銀行合作處理剩餘食材，讓服務更具永續性。

4. 實踐指南：賦賦予地圖生命的「符號」與「視角」

4.1 符號的妙用：箭頭與閃電

* [ ] 箭頭 (Arrows)： 標註互動方向。單向表示資訊發送；雙向則表示「對話」（如：使用者與客服詢問過敏源的來往過程）。
* [ ] 閃電符號 (Lightning Bolts)： 這是最重要的診斷工具。用來標註流程中的「缺口」或「痛點」。當你畫下閃電時，這代表該處是**「需要系統化」或「引進新科技」**來提升效率的關鍵點。

4.2 核心修煉：設計師的專業素養

掌握工具後，你需要具備以下三種思維修煉，才能打破部門孤島 (Silos)：

1. 拒絕「孤狼式」設計： 製作地圖時，必須邀請不同部門的專家與利害關係人共同腦力激盪，才能看見單打獨鬥時忽略的關係。
2. 主動尋找消失的元素： 檢查地圖中是否有被忽視的系統、地點或部門？將它們納入考量，決策才會更完整。
3. 以系統化追求永續： 利用藍圖認清哪些手動流程該被數位系統取代，並思考如何透過跨界合作（如食物銀行）解決浪費，提升社會價值。

5. 總結與學習清單

服務設計的核心在於「全元素的協作」。無論是宏觀的生態系還是微觀的藍圖，最終目的都是要讓我們確保所有元素在對的時間、對的位置共同產出價值。

在開始練習你的專案前，請對照以下檢查清單：

* [ ] 定義中心位： 我是否已清楚設定「C位」並意識到中心改變後的視角差異？
* [ ] 全元素盤點： 地圖是否包含人、組織、地點、部門與系統（如廣告算法）？
* [ ] 辨識交換價值： 是否標記出資訊、數據、貨品、金錢與勞力的流向？
* [ ] 層級劃分： 藍圖是否具備互動線、可見線與內部互動線？
* [ ] 科技層面標註： 我是否在互動線下加入了「科技層面」子類別來應對數位服務？
* [ ] 定位系統化機會： 我是否使用了「閃電符號」標註缺口，並思考如何引進科技或系統化來改善效率？
* [ ] 納入外部系統： 藍圖底部是否包含第三方系統（金流、Email 系統）或外部合作對象（食物銀行）？

透過這些工具，你將能透視隱形的脈絡，設計出更精準、更具溫度的服務體驗！
