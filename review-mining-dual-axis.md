# 評論探勘雙軸評分框架

## 問題背景

`stp-draft.md` 與 `review-mining-scoring.md` 各定義了一套評分規則，但量測的是完全不同的構念：

| | stp-draft（語意強度） | review-mining-scoring（品質評價） |
|---|---|---|
| 尺度 | 0–7 | 0–10 |
| 量測構念 | **Salience（顯著度）**：評論者有沒有提到、提了多少 | **Valence（效價）**：評論者覺得好還是爛 |
| 0 分意義 | 完全沒提到這件事 | 提到了，而且覺得爛透 |
| 滿分意義 | 大篇幅強烈表達 | 提到了，而且覺得極優 |
| 統計用途 | 找消費者關注焦點（定位分析） | 找消費者滿意 / 不滿意面向（品質診斷） |

這兩個維度**不可互換**——一則評論可以對某屬性「講很多但給負評」或「只帶一句但態度正面」。合併成單一分數會造成資訊損失。

---

## 雙軸定義

### Axis 1：Salience（顯著度） 0–7

沿用 `stp-draft.md` 與 `review-mining-stp` SKILL 的既有定義。

| 分數 | 語意 | 判定邏輯 |
|---|---|---|
| 0 | 完全未提及 | 整則評論沒有任何詞彙或語意與該題項相關 |
| 1–2 | 間接暗示 | 用了相關但非直接的詞，或只在附帶子句中帶過 |
| 3 | 輕微提及 | 明確出現關鍵詞但篇幅極短（≤ 1 短句） |
| 4 | 中性帶過 | 提到了但語氣平淡、沒有情緒加重或細節展開 |
| 5 | 明確提及 | 用了具體描述或佐證，佔評論可辨識段落 |
| 6 | 詳細闡述 | 包含多句描述、比較、或具體數據 |
| 7 | 強烈且完整表達 | 該題項是整則評論的核心主題，篇幅與情緒強度最高 |

### Axis 2：Valence（效價） 0–10

沿用 `review-mining-scoring.md` 的品質好壞量尺。

| 分數 | 語意 | 判定邏輯 |
|---|---|---|
| 0 | 極差 | 明確表達該面向徹底失敗、完全不可接受 |
| 1–2 | 很差 | 嚴重不滿，使用強烈負面語詞 |
| 3–4 | 偏差 | 不滿意但語氣較溫和，或有條件性批評 |
| 5 | 中立 | 沒有明顯正負傾向，或正負相抵 |
| 6–7 | 偏好 | 明確正面但沒有特別強調 |
| 8–9 | 很好 | 高度正面，使用肯定語詞或具體稱讚 |
| 10 | 極優 | 毫無保留的讚美，視為該面向的標竿 |

### 交互規則

- 當 `Salience = 0` 時，`Valence` 標記為 `N/A`（沒提到就沒有好壞可言）。
- 當 `Salience ≥ 1` 時，`Valence` 必須給分。

## 雙軸交叉解讀矩陣

|  | Valence 低 (0–3) | Valence 中 (4–6) | Valence 高 (7–10) |
|---|---|---|---|
| **Salience 高 (5–7)** | 🔴 **痛點警報**：消費者大量抱怨，急需改善 | ⚠️ **觀望區**：被大量討論但評價分歧 | 🟢 **核心優勢**：消費者主動宣傳的賣點 |
| **Salience 中 (3–4)** | 需關注：已有負面聲量但尚未爆發 | 一般：不是當前焦點 | 潛力優勢：正面但消費者不太主動提 |
| **Salience 低 (0–2)** | 隱藏風險：少數人提但觀感很差 | 無感：幾乎沒人在意 | 隱藏亮點：提到的人都說好但太少人提 |

## 範例：汽車渦輪增壓器評論

### 範例 1：高 Salience × 低 Valence（痛點）

> **評論原文**：「等了快三個禮拜才收到貨，中間問了兩次客服都沒回覆，最後到的時候外箱壓扁了一角，裡面保麗龍碎一地。幸好渦輪本體沒壞，不然真的要翻臉。這個出貨品質我沒辦法接受。」

| 題項 | Salience | Valence | 理由 |
|---|---|---|---|
| 運送快速 | 6 | 1 | 具體描述等了三週，語氣強烈不滿 |
| 包裝品質 | 7 | 2 | 用了大段篇幅描述外箱壓扁、保麗龍碎裂，是評論核心 |
| 容易溝通 | 5 | 1 | 明確提到問了兩次沒回覆 |
| 品質良好 | 3 | 6 | 提到「渦輪本體沒壞」但只是順帶一提 |

### 範例 2：低 Salience × 高 Valence（隱藏亮點）

> **評論原文**：「裝上去跑了一趟山路，渦輪 2500 轉就開始推，中段加速比原廠好非常多。整體很滿意。對了包裝也蠻用心的。」

| 題項 | Salience | Valence | 理由 |
|---|---|---|---|
| 品質良好 | 6 | 9 | 用轉速和路測結果具體說明效能，高度正面 |
| 容易安裝 | 0 | N/A | 完全沒提到安裝過程 |
| 包裝品質 | 2 | 8 | 只帶一句「蠻用心的」，篇幅極短但態度正面 |
| 高CP值 | 0 | N/A | 未提及價格或性價比 |

### 範例 3：高 Salience × 高 Valence（核心賣點）

> **評論原文**：「價格只有正廠的三分之一，但裝上去增壓效果幾乎一樣，朋友的車裝正廠的我們還拿去比過，體感差異不大。這個 CP 值真的沒話說，已經推薦給車隊裡另外三個人了。」

| 題項 | Salience | Valence | 理由 |
|---|---|---|---|
| 高CP值 | 7 | 10 | 整則評論的核心論點，用價差比較、實測對比、推薦行為三重佐證 |
| 品質良好 | 5 | 8 | 與正廠做了具體比較，結論為「差異不大」 |
| 再購意願 | 4 | 9 | 推薦他人是間接但明確的再購信號 |
| 外觀設計 | 0 | N/A | 未提及 |

### 範例 4：中 Salience × 混合 Valence（觀望區）

> **評論原文**：「尺寸跟原廠的可以直上，不用改管路這點很方便。但聲音比較大，冷車的時候有明顯的金屬摩擦聲，跑熱之後就好了。不確定長期用會不會有問題。」

| 題項 | Salience | Valence | 理由 |
|---|---|---|---|
| 尺寸適配 | 5 | 9 | 明確說可以直上、不用改管路 |
| 品質良好 | 5 | 4 | 有正有負——冷車金屬聲 vs 跑熱正常，語氣偏保留 |
| 容易安裝 | 4 | 7 | 「不用改管路」間接反映安裝容易，但不是主要描述對象 |
| 耐用 | 3 | 4 | 「不確定長期會不會有問題」表達疑慮但沒有定論 |

## 輸出資料表規格

每則評論 × 每個題項一列（長格式），欄位如下：

| 欄位 | 型別 | 說明 |
|---|---|---|
| `review_id` | string | 評論唯一識別碼 |
| `variation` | string | 產品型號 / SKU 分類 |
| `item` | string | 評分題項名稱 |
| `salience` | int 0–7 | 語意強度 |
| `valence` | int 0–10 或 N/A | 品質好壞（salience=0 時為 N/A） |
| `evidence_quote` | string | 對應原文片段（逐字引用） |
| `salience_reason` | string | 為何給這個顯著度分數 |
| `valence_reason` | string | 為何給這個效價分數 |

### 與既有 review-mining-stp SKILL 的相容性

- `review_scoring_table.csv` 的既有 `0–7` 欄位對應本框架的 **Salience 軸**，規格不變。
- **Valence 軸為新增欄位**，以 `_val` 後綴命名（例如 `item_01_val`），或採用長格式獨立欄位。
- `review_foundation.json` 的 `dimension_catalog` 每筆 item 新增 `valence_enabled: true` 標記。

## 品質控管

1. **校準集**：正式跑分前先取 30–50 則評論做人工雙軸評分，作為 golden set。
2. **一致性門檻**：AI 評分與人工評分的 Cohen's Kappa ≥ 0.70（Salience 與 Valence 分開計算）。
3. **易混判準**：
   - 「效率 vs 效能」：效率 = 完成速度，效能 = 完成品質。
   - 「友善 vs 容易溝通」：友善 = 態度，容易溝通 = 回覆速度與資訊清晰度。
   - 「符合預期 vs 與描述一致」：符合預期 = 主觀期望，與描述一致 = 客觀對照賣場文案。
4. **邊界情境**：
   - 諷刺語氣（e.g.「品質真的很『好』呢」）→ Salience 照提及程度給分，Valence 依真實語意判負。
   - 比較句（e.g.「比上一顆好但還是不夠好」）→ Valence 依整體結論方向，不取單一子句。
